This  provides a clear and practical explanation of **Network Address Translation (NAT)** , one of the most critical technologies enabling the continued use of IPv4 despite address exhaustion. It covers the fundamentals of NAT, the distinction between private and public IP addresses (RFC 1918), and the more efficient **NAT Overload (PAT)** mechanism.


---

### Network Address Translation (NAT) Cheat Sheet

This guide covers the purpose of NAT, the private IP address ranges defined in RFC 1918, and how both standard NAT and NAT Overload (PAT) work.

---

#### The Problem: IPv4 Address Exhaustion

| Fact | Implication |
| :--- | :--- |
| IPv4 has approximately 4.29 billion addresses. | Not enough for the 20–30 billion+ devices connected to the internet. |
| All IPv4 address blocks have been exhausted. | No new public IPv4 addresses are available for assignment. |
| Private IP addresses (RFC 1918) are not routable on the internet. | A mechanism is needed to allow private devices to communicate with public internet resources. |

---

#### Private IP Address Ranges (RFC 1918)

These IP address ranges are reserved for private use. They are **not routable on the public internet** and can be reused across different organizations.

| Network Range | CIDR Notation | Address Count | Common Use Case |
| :--- | :--- | :--- | :--- |
| `10.0.0.0` – `10.255.255.255` | `10.0.0.0/8` | ~16.7 million | Large enterprise networks |
| `172.16.0.0` – `172.31.255.255` | `172.16.0.0/12` | ~1 million | Medium-sized networks |
| `192.168.0.0` – `192.168.255.255` | `192.168.0.0/16` | ~65,000 | Home networks, small businesses |

---

#### What is NAT?

**Network Address Translation (NAT)** is a technology that translates IP addresses between different networks—typically between private (internal) addresses and public (external) addresses.

| Aspect | Description |
| :--- | :--- |
| **Primary Purpose** | Allows devices with private IP addresses to communicate with the public internet using a public IP address. |
| **How It Works** | A NAT-enabled router modifies the source IP address of outgoing packets and maintains a translation table to reverse the process for incoming replies. |
| **Benefits** | Conserves public IP addresses; adds a layer of obscurity (internal addresses are hidden). |

---

#### How NAT Works (Basic Translation)

**Scenario:** Vala (`10.10.20.50`) wants to reach `professormesser.com` (`104.20.19.63`).

| Step | Direction | Source IP | Destination IP | Action |
| :--- | :--- | :--- | :--- | :--- |
| **1** | Outbound | `10.10.20.50` | `104.20.19.63` | Vala sends packet to NAT router. |
| **2** | Translation | `94.1.1.1` | `104.20.19.63` | NAT router changes source IP to public address. |
| **3** | Internet | `94.1.1.1` | `104.20.19.63` | Packet traverses internet. |
| **4** | Response | `104.20.19.63` | `94.1.1.1` | Web server replies to public IP. |
| **5** | Reverse Translation | `104.20.19.63` | `10.10.20.50` | NAT router looks up translation table and forwards to Vala. |

**Limitation:** With standard NAT, each internal device requires a unique public IP address for outbound connections. This does not solve address exhaustion.

---

#### NAT Overload (PAT)

**NAT Overload**, also known as **Port Address Translation (PAT)** , is the most common form of NAT. It allows multiple internal devices to share a single public IP address by using unique port numbers to track sessions.

| Concept | Description |
| :--- | :--- |
| **Mechanism** | Translates both the IP address and the source port number. |
| **Efficiency** | Many internal devices can use the same public IP address simultaneously. |
| **Table Entry** | Each session is tracked by a combination of (private IP:port) → (public IP:port). |

**Example: Two Users Sharing One Public IP**

| User | Private Address:Port | Translated to (Public:Port) |
| :--- | :--- | :--- |
| Vala | `10.10.20.50:3233` | `94.1.1.1:1055` |
| Jonas | `10.10.20.70:5782` | `94.1.1.1:1056` |

Both users share the same public IP (`94.1.1.1`) but are distinguished by the port number. The NAT router maintains a table to map incoming replies back to the correct internal device.

---

#### NAT Translation Table (PAT Example)

| Protocol | Private Address | Private Port | Public Address | Public Port | Destination | Destination Port |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| TCP | `10.10.20.50` | 3233 | `94.1.1.1` | 1055 | `104.20.19.63` | 80 |
| TCP | `10.10.20.70` | 5782 | `94.1.1.1` | 1056 | `104.20.19.63` | 80 |

When a reply arrives from `104.20.19.63:80` to `94.1.1.1:1055`, the router consults the table, sees that this maps to Vala's private IP and port, and forwards the packet accordingly.

---

#### Types of NAT

| Type | Description | Use Case |
| :--- | :--- | :--- |
| **Static NAT** | One-to-one mapping between a private IP and a public IP. | Hosting a public-facing server internally (e.g., web server). |
| **Dynamic NAT** | Maps a private IP to a public IP from a pool; no guaranteed consistency. | Outbound connections where public IPs are available but limited. |
| **NAT Overload (PAT)** | Maps multiple private IPs to a single public IP using port numbers. | Most home and business networks; enables many devices to share one public IP. |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Private IPs Are Not Routable** | RFC 1918 addresses cannot traverse the internet without translation. |
| **NAT Enables Internet Access** | NAT translates private source IPs to public IPs for outbound communication. |
| **PAT Conserves Addresses** | By using port numbers, thousands of internal devices can share a single public IP. |
| **NAT Is Not Security** | While NAT hides internal addressing, it is not a substitute for a firewall. |
| **Common in Every Home** | The home router you use performs PAT to allow all your devices to share one ISP-provided public IP. |
