### Routing Table Fundamentals Cheat Sheet

This guide covers the components of a routing table, how routers select the best path, and advanced concepts like subinterfaces and router redundancy.

---

#### Routing Table Overview

A routing table is a data structure used by routers (and any IP-enabled device) to determine where to forward packets. Every device with an IP stack has a routing table.

| Component | Description |
| :--- | :--- |
| **Route Code** | Indicates how the route was learned (e.g., C = directly connected, S = static, R = RIP, D = EIGRP, O = OSPF). |
| **Destination Network** | The network address and prefix length (e.g., `10.0.30.0/24`). |
| **Administrative Distance** | A value that indicates the trustworthiness of the route source. Lower = better. |
| **Metric** | A protocol-specific value used to choose between multiple routes from the same source. Lower = better. |
| **Next Hop** | The IP address of the next router to forward the packet to. |
| **Outgoing Interface** | The physical or logical interface to use when forwarding the packet. |

---

#### Route Selection Process

When a router receives a packet, it follows a three-step process to determine the best path:

| Step | Factor | Description |
| :--- | :--- | :--- |
| **1** | **Longest Prefix Match** | Among all routes that match the destination IP, the route with the **longest prefix length** (most specific) is preferred. |
| **2** | **Administrative Distance (AD)** | If multiple routes have the same prefix length, the route with the **lowest AD** is preferred. This compares routes from different sources (e.g., static vs. OSPF). |
| **3** | **Metric** | If multiple routes from the same protocol have the same prefix length and AD, the route with the **lowest metric** is preferred. This compares routes from the same source (e.g., two OSPF routes). |

---

#### Longest Prefix Match

When multiple routes match a destination IP address, the router chooses the most specific route (the one with the longest prefix length).

**Example:** Destination IP = `192.168.1.6`

| Route | Prefix Length | Specificity | Choice |
| :--- | :--- | :--- | :--- |
| `192.168.0.0/16` | 16 bits | Least specific | ❌ |
| `192.168.1.0/24` | 24 bits | More specific | ❌ (unless /32 not present) |
| `192.168.1.6/32` | 32 bits | Most specific (host route) | ✅ |

**Key Takeaway:** A `/32` host route always wins over any less specific network route.

---

#### Administrative Distance (AD)

Administrative Distance is a value that indicates the trustworthiness of a route source. Lower values are preferred. AD is used when the same destination is learned from different routing sources.

| Route Source | Administrative Distance (Cisco Default) |
| :--- | :--- |
| Directly Connected | 0 |
| Static Route | 1 |
| EIGRP Summary Route | 5 |
| External BGP (eBGP) | 20 |
| EIGRP (Internal) | 90 |
| OSPF | 110 |
| RIP | 120 |
| External EIGRP | 170 |
| Internal BGP (iBGP) | 200 |
| Unknown / Unusable | 255 |

**Example:** If a router learns the same network via both OSPF (AD 110) and RIP (AD 120), it will choose the OSPF route because it has the lower AD.

---

#### Metric

A metric is a protocol-specific value used to choose between multiple routes from the **same routing protocol**. Lower metrics are preferred.

| Protocol | Metric Type | Description |
| :--- | :--- | :--- |
| **RIP** | Hop Count | Number of routers between source and destination. |
| **OSPF** | Cost | Based on interface bandwidth (10^8 / bandwidth). |
| **EIGRP** | Composite | Bandwidth, delay, load, reliability, MTU (bandwidth + delay are primary). |
| **BGP** | Multiple Attributes | AS-Path length, Local Preference, MED, etc. |

**Important:** Metrics cannot be compared across different protocols. A RIP metric of 1 is not "better" than an OSPF metric of 50. The AD is used to choose between protocols; metrics are used within the same protocol.

---

#### Subinterfaces

A **subinterface** is a virtual interface created on a physical interface, allowing a single physical connection to carry traffic for multiple VLANs or networks.

| Concept | Description |
| :--- | :--- |
| **Use Case** | Router-on-a-stick (connecting a router to a trunk port on a switch). |
| **Syntax** | Physical interface `g0/0` with subinterfaces `g0/0.10`, `g0/0.20`, etc. |
| **Benefits** | Allows one physical interface to route between multiple VLANs. Each subinterface can have its own IP address, subnet mask, and security policies. |

**Example Configuration:**
```
Physical Interface: g0/0 (trunk to switch)
  ├── Subinterface g0/0.1  → VLAN 10 → IP 10.10.10.1/24
  ├── Subinterface g0/0.2  → VLAN 20 → IP 10.10.20.1/24
  └── Subinterface g0/0.3  → VLAN 30 → IP 10.10.30.1/24
```

#### First-Hop Redundancy Protocols (FHRP)

FHRP provides router redundancy for end devices that have a single default gateway configured.

| Component | Description |
| :--- | :--- |
| **Problem** | End devices can only have one default gateway IP configured. |
| **Solution** | A **virtual IP address (VIP)** is shared between multiple routers. End devices use this VIP as their default gateway. |
| **Mechanism** | Active router handles all traffic. Standby router monitors the active router. If the active fails, the standby takes over the VIP. |

**Common FHRP Protocols:**
| Protocol | Full Name | Description |
| :--- | :--- | :--- |
| **HSRP** | Hot Standby Router Protocol | Cisco-proprietary; uses active/standby model. |
| **VRRP** | Virtual Router Redundancy Protocol | Open standard; similar to HSRP. |
| **GLBP** | Gateway Load Balancing Protocol | Cisco-proprietary; provides load balancing across multiple routers. |

**How It Works:**
1. Router A (Active) and Router B (Standby) share a virtual IP (e.g., `192.168.1.1`).
2. End devices use `192.168.1.1` as their default gateway.
3. Router A handles all traffic.
4. If Router A fails, Router B detects the failure and assumes the virtual IP.
5. End users experience no interruption.

#### Routing Table Example (RIP)

```
R    10.10.30.0/24 [120/1] via 10.10.50.2, 00:00:14, Serial0/3/1
```

| Field | Value | Meaning |
| :--- | :--- | :--- |
| Route Code | `R` | Learned via RIP |
| Destination | `10.10.30.0/24` | Target network |
| AD | `120` | RIP's default AD |
| Metric | `1` | One hop away |
| Next Hop | `10.10.50.2` | Next router IP |
| Timestamp | `00:00:14` | Route active for 14 seconds |
| Interface | `Serial0/3/1` | Outgoing interface |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Longest Prefix Match First** | The router always chooses the most specific matching route. |
| **AD Chooses Between Protocols** | Administrative distance compares routes from different sources. Lower = better. |
| **Metric Chooses Within Protocols** | Metrics compare routes from the same source. Lower = better. |
| **Subinterfaces Enable Router-on-a-Stick** | A single physical interface can route between multiple VLANs. |
| **FHRP Provides Redundancy** | Virtual IP allows seamless failover between routers without changing end-user configuration. |
