### Wireless Network Types and Modes Cheat Sheet

This guide covers the four primary wireless network configurations: mesh, ad hoc, point-to-point, and infrastructure mode.

---

#### Comparison of Wireless Modes

| Mode | Also Known As | Description | Use Case | Requires Access Point? |
| :--- | :--- | :--- | :--- | :--- |
| **Infrastructure Mode** | BSS (Basic Service Set) | Devices connect to a central access point, which bridges traffic to the wired network. | Home Wi-Fi, office networks, public hotspots. | **Yes** |
| **Ad Hoc Mode** | IBSS (Independent Basic Service Set) | Devices connect directly to each other without an access point. | Temporary file transfers, IoT device setup (e.g., configuring a smart bulb). | **No** |
| **Point-to-Point Mode** | Wireless Bridge | Two access points connect directly to each other over a distance, linking two separate networks. | Connecting two buildings (homes, offices) across a distance without running cable. | **Yes (two APs, no central controller)** |
| **Wireless Mesh** | Mesh Network | Multiple access points communicate with each other wirelessly, forming a self-healing, redundant network. | Large homes, outdoor areas, campuses where cabling every AP is impractical. | **Yes (multiple APs meshed)** |

---

#### Infrastructure Mode

This is the most common wireless configuration.

| Aspect | Description |
| :--- | :--- |
| **Architecture** | One or more access points (APs) connected to a wired network. Wireless clients (laptops, phones) connect to the AP. |
| **Client Isolation** | Some APs allow client isolation, where wireless devices cannot communicate directly with each other—only to the wired network (useful for public hotspots). |
| **Typical Environment** | Home routers (combined AP/router/switch), enterprise deployments with multiple APs and a controller. |

---

#### Ad Hoc Mode (IBSS)

Ad hoc mode enables direct device-to-device communication without any infrastructure.

| Aspect | Description |
| :--- | :--- |
| **Architecture** | Two or more devices communicate directly over 802.11 wireless. No access point involved. |
| **Temporary Use** | Often used for short-term connections, such as transferring a file or setting up an IoT device. |
| **IoT Setup Example** | 1. Device (e.g., smart plug) creates an ad hoc network. <br> 2. Phone connects to that ad hoc network. <br> 3. Phone transfers home Wi-Fi credentials to the device. <br> 4. Device disconnects ad hoc and connects to the home Wi-Fi. |

---

#### Point-to-Point Mode (Wireless Bridge)

Point-to-point mode connects two separate networks wirelessly over a distance.

| Aspect | Description |
| :--- | :--- |
| **Architecture** | Two access points configured in point-to-point mode, each connected to its own local wired network. The APs communicate directly with each other, bridging the two networks. |
| **Distance** | Can cover significant distances, especially with directional antennas and higher transmit power. |
| **Requirements** | Both APs must support point-to-point mode. May require specialized antennas (Yagi, parabolic) and power adjustments. |
| **Example** | House A (with AP and switch) wirelessly connected to House B (with AP and switch), creating a single logical network across both properties. |

---

#### Wireless Mesh

A wireless mesh network consists of multiple access points that communicate with each other wirelessly, providing redundancy and extended coverage without cabling every AP.

| Aspect | Description |
| :--- | :--- |
| **Architecture** | Multiple APs form a mesh backhaul. One or more APs are wired to the network (root APs); others connect wirelessly through neighboring mesh APs. |
| **Self-Healing** | If one AP fails or is removed, traffic automatically reroutes through other mesh APs. |
| **Scalability** | Adding more APs extends coverage and can improve mesh density. |
| **Use Cases** | Large homes (e.g., Google Nest Wi-Fi, Eero), outdoor areas, warehouses, campuses where running Ethernet to every AP is difficult. |

**Note:** Consumer mesh systems often have a dedicated wireless backhaul band (e.g., 5 GHz or 6 GHz) for AP-to-AP communication, separate from the band used by client devices.

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Infrastructure Mode is the Standard** | Most home and office Wi-Fi uses a central access point. |
| **Ad Hoc Enables Direct Device-to-Device** | No AP required; useful for temporary connections and IoT provisioning. |
| **Point-to-Point Connects Two Networks** | Two APs bridge separate wired networks across a distance (e.g., building to building). |
| **Mesh Provides Redundancy and Extended Coverage** | APs communicate wirelessly with each other, self-healing when a node fails. |
| **Each Mode Serves a Different Purpose** | Choose based on scale, distance, need for cabling, and whether a central AP exists. |
