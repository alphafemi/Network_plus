### Wireless Networking Architecture Cheat Sheet

This guide covers wireless network topologies, identifiers, security modes, antenna types, and management approaches.

---

#### Wireless Network Topologies

| Topology | Also Known As | Description | Use Case |
| :--- | :--- | :--- | :--- |
| **IBSS (Independent Basic Service Set)** | **Ad Hoc** | Devices connect directly to each other without an access point. | IoT device setup (e.g., configuring a smart doorbell), temporary file sharing between two devices. |
| **BSS (Basic Service Set)** | Infrastructure Mode | Devices connect to a single access point. | Home Wi-Fi, small office with one AP. |
| **ESS (Extended Service Set)** | Multiple APs, Same SSID | Multiple access points share the same network name (SSID), enabling seamless roaming. | Enterprise networks, large homes with multiple APs, campuses. |

---

#### Wireless Identifiers: SSID vs. BSSID vs. ESSID

| Identifier | Full Name | Description |
| :--- | :--- | :--- |
| **SSID** | Service Set Identifier | The **human-readable name** of a wireless network (e.g., "CoffeeShop Wi-Fi"). |
| **BSSID** | Basic Service Set Identifier | The **MAC address** of a specific access point. Used to differentiate between APs sharing the same SSID. |
| **ESSID** | Extended Service Set Identifier | The shared SSID used across multiple access points in an extended network, enabling roaming. |

**Example:**
- You see a network named **SGC1** (the SSID/ESSID).
- Behind the scenes, AP1 has BSSID `60:3D:26:11:22:33`, AP2 has BSSID `60:3D:26:44:55:66`.
- Your device roams between them seamlessly because they share the same ESSID.

#### Captive Portal

A **captive portal** is a web page presented to users before they gain full network access. It is commonly used in public Wi-Fi, hotels, and guest networks.

| Feature | Description |
| :--- | :--- |
| **Function** | Intercepts initial web requests and redirects to a login/acceptance page. |
| **Authentication** | May require username/password, acceptance of terms, or a voucher code. |
| **Access Table** | Maintains a list of authenticated devices; once authenticated, access is granted for a defined period (e.g., 24 hours). |

#### Wireless Security Modes

| Security Mode | Also Known As | Description | Common Use |
| :--- | :--- | :--- | :--- |
| **Open** | None | No security; anyone can connect. | Public hotspots (with captive portal for authentication). |
| **WEP** | Wired Equivalent Privacy | Legacy, easily cracked. Should not be used. | Obsolete. |
| **WPA/WPA2-Personal** | WPA-PSK (Pre-Shared Key) | Uses a shared password for all users. | Home networks, small businesses. |
| **WPA/WPA2-Enterprise** | 802.1X | Uses individual credentials (username/password or certificates). Authenticates against a RADIUS server. | Enterprise networks, schools, corporations. |
| **WPA3-Personal** | WPA3-PSK | Enhanced security over WPA2; uses Simultaneous Authentication of Equals (SAE) to resist offline dictionary attacks. | Modern home and small business networks. |
| **WPA3-Enterprise** | WPA3-802.1X | Enterprise-grade security with enhanced encryption and authentication. | Enterprise networks requiring highest security. |
| **OWE** | Opportunistic Wireless Encryption | Encrypts the connection between client and AP without requiring authentication. Prevents passive eavesdropping on open networks. | Open networks (e.g., public Wi-Fi) that need basic privacy. |

#### Antenna Types

| Antenna Type | Radiation Pattern | Description | Use Case |
| :--- | :--- | :--- | :--- |
| **Omnidirectional** | 360° horizontal, donut-shaped | Sends and receives signal evenly in all directions. | Home APs, placing AP in the center of coverage area. |
| **Directional** | Focused beam | Concentrates signal in a specific direction. | Long-distance links, covering hallways, connecting buildings. |

**Directional Antenna Examples:**
- **Yagi:** Highly directional, high gain. Used for point-to-point links.
- **Parabolic (dish):** Focuses signal to a single point. Used for very long distances (e.g., connecting buildings across a campus).

**Gain (dB):** Measured in decibels. Every **3 dB of gain doubles the effective power** of the signal.

#### Access Point Management Models

| Model | Description | Use Case |
| :--- | :--- | :--- |
| **Autonomous AP** | Standalone access point. Configured individually via web interface or CLI. | Home networks, small offices with a single AP. |
| **Lightweight AP (LAP)** | AP handles only real-time radio functions; control and configuration are managed centrally. | Enterprise networks with many APs. |
| **Wireless LAN Controller (WLC)** | Centralized device that manages all lightweight APs. Provides a **single pane of glass** for configuration, monitoring, and reporting. | Large enterprises, campuses, multi-site deployments. |

**CAPWAP (Control and Provisioning of Wireless Access Points):**
- The standard protocol used between lightweight APs and the WLC.
- Allows the controller to manage AP configuration, firmware, and security policies centrally.

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Ad Hoc (IBSS) is Direct Device-to-Device** | Used for temporary connections without an AP (e.g., IoT setup). |
| **ESS Enables Roaming** | Multiple APs sharing the same SSID allow seamless movement across a large area. |
| **SSID is the Network Name; BSSID is the AP's MAC** | You see the SSID; the BSSID identifies the specific AP behind the scenes. |
| **Captive Portals Authenticate Guest Users** | Common in public Wi-Fi; intercepts traffic until the user accepts terms or logs in. |
| **WPA2/WPA3-Enterprise Uses Individual Credentials** | Unlike PSK (shared password), enterprise mode uses 802.1X for per-user authentication and revocation. |
| **Antenna Choice Affects Coverage** | Omnidirectional for 360° coverage; directional for focused, long-range links. |
| **Lightweight APs + WLC Scale Wireless** | Enterprise Wi-Fi uses centralized controllers (via CAPWAP) to manage many APs from one interface. |
