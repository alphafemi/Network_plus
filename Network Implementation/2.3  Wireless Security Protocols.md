### Wireless Security Protocols Cheat Sheet

This guide covers the evolution of Wi-Fi security, from WEP to WPA3, explaining the goals of wireless security and the cryptographic methods used.

---

#### The Three Goals of Wireless Security

Wireless networks must address three core security objectives to protect data in transit.

| Goal | Description |
| :--- | :--- |
| **Authentication** | Verifying the identity of the user or device attempting to connect to the network. Methods include PSK (shared password), 802.1X (username/password or certificate), or multi-factor authentication. |
| **Confidentiality (Encryption)** | Ensuring that data transmitted over the air cannot be read by unauthorized parties. Achieved through encryption protocols. |
| **Message Integrity** | Verifying that the data received has not been altered or tampered with during transmission. |

---

#### Wireless Security Protocol Evolution

| Protocol | Full Name | Introduced | Status | Key Cryptographic Method | Notes |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **WEP** | Wired Equivalent Privacy | 1999 | **Broken / Obsolete** | RC4 stream cipher | Significant cryptographic vulnerabilities; can be cracked in minutes. Should never be used. |
| **WPA** | Wi-Fi Protected Access | 2003 | **Deprecated** | TKIP (Temporal Key Integrity Protocol) | Designed as a stopgap to work on WEP hardware. Improved over WEP but still has vulnerabilities. |
| **WPA2** | Wi-Fi Protected Access 2 | 2004 | **Still widely used, but being replaced** | **CCMP** (AES + CBC-MAC) | Introduced AES encryption and combined encryption/integrity. The standard for over a decade. |
| **WPA3** | Wi-Fi Protected Access 3 | 2018 | **Current standard / Recommended** | **GCMP** (AES + GMAC) | Enhanced security for personal and enterprise modes; protects against offline dictionary attacks (SAE). |

---

#### Cryptographic Protocols Explained

| Protocol | Full Name | Encryption | Integrity | Used In |
| :--- | :--- | :--- | :--- | :--- |
| **TKIP** | Temporal Key Integrity Protocol | RC4 (with per-packet key mixing) | MIC (Michael) | WPA |
| **CCMP** | Counter Mode CBC-MAC Protocol | **AES** (Advanced Encryption Standard) | **CBC-MAC** (Cipher Block Chaining Message Authentication Code) | WPA2 |
| **GCMP** | Galois Counter Mode Protocol | **AES** (Advanced Encryption Standard) | **GMAC** (Galois Message Authentication Code) | WPA3 |

**Key Differences:**
- **CCMP** and **GCMP** both use AES for strong encryption.
- **GCMP** (used by WPA3) is more efficient and provides stronger integrity protection than CCMP.
- **WPA3** also introduces **SAE (Simultaneous Authentication of Equals)** for personal mode, replacing WPA2's PSK (Pre-Shared Key), which is vulnerable to offline dictionary attacks.

---

#### Security Mode Comparison

| Mode | Authentication Method | Suitable For |
| :--- | :--- | :--- |
| **WPA2/WPA3-Personal** | Pre-Shared Key (PSK) – a single shared password. WPA3 uses SAE for stronger protection. | Home networks, small offices. |
| **WPA2/WPA3-Enterprise** | 802.1X with RADIUS server – individual credentials (username/password or certificates). | Enterprises, schools, organizations requiring per-user authentication and access control. |

---

#### Recommendations

| Scenario | Recommended Security |
| :--- | :--- |
| **New devices** | **WPA3** (Personal or Enterprise, depending on environment) |
| **Mixed environment (older clients)** | WPA2 with CCMP (AES) – avoid TKIP if possible |
| **Avoid at all costs** | WEP, Open networks (without additional security like captive portal + VPN) |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **WEP is completely broken** | Should never be used on any modern network. |
| **WPA2 (AES-CCMP) is still acceptable** | Widely supported and secure when properly configured, though WPA3 is preferred. |
| **WPA3 is the future** | Provides stronger protections, including SAE to resist offline dictionary attacks. |
| **Enterprise mode is more secure than Personal** | Individual credentials can be revoked; PSK cannot be changed without updating every device. |
| **Wireless security requires all three goals** | Authentication, confidentiality (encryption), and integrity must all be addressed. |
