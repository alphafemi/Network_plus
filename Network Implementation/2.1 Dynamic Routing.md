### Dynamic Routing Cheat Sheet

This guide covers the fundamentals of dynamic routing, the benefits over static routing, and the characteristics of the major dynamic routing protocols.

---

#### Static vs. Dynamic Routing

| Aspect | Static Routing | Dynamic Routing |
| :--- | :--- | :--- |
| **Configuration** | Manual configuration of every route on every router. | Initial configuration of the routing protocol; routers share routes automatically. |
| **Scalability** | Impractical for networks with more than a handful of routers. | Scales to hundreds or thousands of routers. |
| **Adaptability** | Does not adapt to network changes. | Automatically updates when routers are added, removed, or links fail. |
| **Overhead** | No CPU/memory overhead. | Uses CPU, memory, and some network bandwidth for routing updates. |
| **Failover** | No automatic failover. | Can reroute traffic around failed links automatically. |

---

#### How Dynamic Routing Works

Dynamic routing protocols automate the process of building and maintaining routing tables through a continuous cycle:

| Step | Description |
| :--- | :--- |
| **1. Discovery** | Routers listen for routing updates from neighboring routers (unicast or multicast). |
| **2. Exchange** | Routers share their known routes with other routers. |
| **3. Decision** | Each router evaluates incoming routes and updates its routing table if a better path is found (based on the protocol's metric). |
| **4. Convergence** | After a topology change, routers propagate updates until all routers have consistent, accurate routing information. |

#### Dynamic Routing Protocol Comparison

| Protocol | Full Name | Type | Key Characteristics | Best For |
| :--- | :--- | :--- | :--- | :--- |
| **EIGRP** | Enhanced Interior Gateway Routing Protocol | Advanced Distance Vector | - Cisco-proprietary (though partially open) <br> - Easy to configure <br> - Fast convergence <br> - Loop prevention built-in <br> - Minimal network overhead | Cisco-centric environments; good for most enterprise networks |
| **OSPF** | Open Shortest Path First | Link-State | - Open standard (vendor-neutral) <br> - Uses cost metric (based on bandwidth) <br> - Fast convergence <br> - Hierarchical design (areas) | Multi-vendor environments; large enterprise networks |
| **BGP** | Border Gateway Protocol | Path-Vector | - Used between autonomous systems (AS) <br> - "The three-napkin protocol" <br> - Path selection based on policies, not just speed <br> - Scales to the entire internet | Internet connectivity; connections between organizations; WAN edge |

---

#### EIGRP (Enhanced Interior Gateway Routing Protocol)

| Aspect | Description |
| :--- | :--- |
| **Type** | Advanced Distance Vector (often called a hybrid protocol). |
| **Vendor** | Originally Cisco-proprietary; now partially open, but still most common in Cisco networks. |
| **Metric** | Composite metric based on bandwidth, delay, load, reliability, and MTU (bandwidth + delay are primary). |
| **Convergence** | Very fast; uses Diffusing Update Algorithm (DUAL) to pre-compute backup routes. |
| **Overhead** | Low; sends updates only when changes occur, not on a fixed schedule. |
| **Configuration** | Relatively simple to set up. |
| **Loop Prevention** | DUAL ensures loop-free paths. |

**When to use EIGRP:**
- You have a Cisco-centric network.
- You want fast convergence and simple configuration.
- You are comfortable with a protocol that is not fully open standard.

---

#### OSPF (Open Shortest Path First)

| Aspect | Description |
| :--- | :--- |
| **Type** | Link-State Protocol. |
| **Vendor** | Open standard (RFC); available on virtually all network devices. |
| **Metric** | **Cost** (typically based on interface bandwidth: 10^8 / bandwidth in bps). Lower cost = better path. |
| **Convergence** | Fast; uses flooding of Link-State Advertisements (LSAs) and the SPF algorithm. |
| **Scalability** | Highly scalable; supports hierarchical design with **areas** (Area 0 is the backbone). |
| **Load Balancing** | Supports equal-cost multi-path (ECMP) for load balancing across identical-cost links. |
| **Configuration** | Requires more planning than EIGRP (area design, router IDs, etc.). |

**When to use OSPF:**
- You have a multi-vendor environment (Cisco, Juniper, Arista, etc.).
- You need an open standard protocol.
- You have a large network that benefits from hierarchical design.

---

#### BGP (Border Gateway Protocol)

| Aspect | Description |
| :--- | :--- |
| **Type** | Path-Vector Protocol (sometimes classified as a distance vector protocol). |
| **Vendor** | Open standard; the core routing protocol of the internet. |
| **Metric** | No single metric; uses a complex decision process (attributes) including **AS-Path length**, **local preference**, **MED**, and others. |
| **Convergence** | Slower than IGP protocols (EIGRP/OSPF); designed for stability over speed. |
| **Use Case** | **Inter-domain routing**—connecting different autonomous systems (e.g., your network to your ISP, or between ISPs). |
| **Internal vs. External** | **eBGP** = between different ASes; **iBGP** = within the same AS. |
| **Nickname** | Sometimes called the "three-napkin protocol" because it was initially sketched on napkins. |

**When to use BGP:**
- You have multiple connections to the internet (multi-homing) and want to influence inbound/outbound traffic.
- You are a service provider or large enterprise connecting to other autonomous systems.
- You need policy-based routing (e.g., "prefer this ISP for traffic to a specific network").

---

#### Routing Protocol Types: Interior vs. Exterior

| Category | Full Name | Purpose | Protocols |
| :--- | :--- | :--- | :--- |
| **IGP** | Interior Gateway Protocol | Routes within a single autonomous system (your organization's network). | OSPF, EIGRP, RIP |
| **EGP** | Exterior Gateway Protocol | Routes between different autonomous systems. | **BGP** (the only EGP in use today) |

**Autonomous System (AS):** A collection of networks under a single administrative control, with a consistent routing policy.

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Dynamic Routing Automates Route Management** | Routers discover, share, and update routes automatically, eliminating manual static route configuration. |
| **Trade-off: Overhead vs. Automation** | Dynamic routing uses CPU, memory, and bandwidth but scales far better than static routing. |
| **EIGRP = Cisco-Friendly** | Easy to configure, fast convergence, but primarily used in Cisco environments. |
| **OSPF = Open Standard** | Works across all vendors; highly scalable with hierarchical areas; ideal for multi-vendor networks. |
| **BGP = Internet Backbone** | Handles routing between autonomous systems; essential for multi-homed internet connections and service provider networks. |
| **Convergence Speed Varies** | EIGRP and OSPF converge quickly (seconds); BGP converges more slowly but prioritizes stability. |
