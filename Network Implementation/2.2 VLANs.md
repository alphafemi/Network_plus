It covers what VLANs are, how they work, how to connect them across switches using trunking, and advanced concepts like native VLANs, layer 3 switching, and voice VLANs.

### VLANs and Trunking Cheat Sheet

This guide covers the purpose of VLANs, how they are configured, how trunking enables VLANs to span multiple switches, and related concepts like native VLANs, layer 3 switching, and voice VLANs.

---

#### What is a VLAN?

A **VLAN (Virtual Local Area Network)** is a logical segmentation of a physical network. Devices in the same VLAN are in the same broadcast domain, regardless of their physical location.

| Concept | Description |
| :--- | :--- |
| **Broadcast Domain** | All devices that receive broadcast traffic from each other. Without VLANs, all devices on a switch share a single broadcast domain. |
| **VLAN** | Divides a single physical switch into multiple isolated broadcast domains. |

**Without VLANs (Physical Separation):**
```
[Switch A - Red]         [Switch B - Blue]
   |     |                 |     |
  PC1   PC2               PC3   PC4
```
- Two separate switches (costly, more management, more rack space)
- Red network cannot communicate with blue network

**With VLANs (Logical Separation on One Switch):**
```
        [Single Switch]
    VLAN 10      VLAN 20
     |    |      |    |
    PC1  PC2    PC3  PC4
```
- One physical switch
- VLAN 10 devices cannot communicate with VLAN 20 devices (unless routed)
- Efficient use of hardware, lower cost, easier management

---

#### VLAN Tagging and 802.1Q

When VLANs need to span multiple switches, a single connection (trunk) can carry traffic for multiple VLANs using **tagging**.

| Concept | Description |
| :--- | :--- |
| **802.1Q (dot1q)** | The IEEE standard for VLAN tagging. Inserts a 4-byte tag into the Ethernet frame after the source MAC address. |
| **VLAN Tag** | Contains a **12-bit VLAN ID**, allowing up to 4,094 VLANs (2^12 = 4,096, minus reserved IDs). |
| **Trunk** | A port configured to carry traffic for multiple VLANs using 802.1Q tagging. |

**How Tagging Works:**
```
Original Frame:
+-----------+-----------+------+---------+--------+
| Dest MAC  | Src MAC   | Type | Payload | FCS    |
+-----------+-----------+------+---------+--------+

Tagged Frame (on trunk):
+-----------+-----------+--------+------+---------+--------+
| Dest MAC  | Src MAC   | VLAN Tag | Type | Payload | FCS    |
+-----------+-----------+--------+------+---------+--------+
                         ^
                         |
                    12-bit VLAN ID
```

**VLAN Trunk Example:**
```
Switch A                    Switch B
VLAN 10 ----+               +---- VLAN 10
VLAN 20 ----+---[Trunk]-----+---- VLAN 20
            (802.1Q)        |
VLAN 30 ----+               +---- VLAN 30
```
- A single cable carries traffic for all VLANs
- Each frame is tagged with its VLAN ID
- Receiving switch removes the tag and forwards to the correct VLAN

#### Default VLAN vs. Native VLAN

| VLAN Type | Description |
| :--- | :--- |
| **Default VLAN** | The VLAN all switch ports belong to when the switch is first powered on. Typically **VLAN 1**. |
| **Native VLAN** | On a trunk port, the VLAN whose traffic is **not tagged**. Used for backward compatibility with devices that don't understand 802.1Q tags. |

**Important:**
- Native VLAN must match on both ends of a trunk; otherwise, VLAN leakage or errors can occur.
- Management traffic (like CDP, VTP) often uses the native VLAN.

#### Layer 3 Switching and SVIs

A **layer 3 switch** combines the functions of a switch (layer 2) and a router (layer 3) in a single device.

| Concept | Description |
| :--- | :--- |
| **Layer 3 Switch** | Can route traffic between VLANs without needing an external router. |
| **SVI (Switched Virtual Interface)** | A virtual interface on a layer 3 switch that acts as the gateway for a VLAN. Each VLAN that needs routing gets an SVI with an IP address. |

**Routing Between VLANs:**
```
Without Layer 3 Switch:          With Layer 3 Switch:
[Switch] --- [Router]            [Layer 3 Switch]
VLAN 10 ------|                  VLAN 10 --- SVI 10 (192.168.10.1)
VLAN 20 ------|                  VLAN 20 --- SVI 20 (192.168.20.1)
```
- The layer 3 switch handles routing internally, saving hardware and simplifying management.

**Note:** Layer 3 switches are generally less capable than dedicated routers for advanced routing features (BGP, complex NAT, etc.). The adage: **"Switch when you can, route when you must."**

---

#### Voice VLANs

Voice over IP (VoIP) requires consistent, low-latency communication. Voice VLANs separate voice traffic from data traffic to ensure quality.

| Concept | Description |
| :--- | :--- |
| **Voice VLAN** | A dedicated VLAN for IP phones, separate from data VLAN. |
| **Physical Setup** | Phone connects to switch port. Computer connects to pass-through port on phone. |
| **Trunking to Phone** | The switch port is configured as a trunk carrying both voice and data VLANs. The phone tags its traffic, the computer's traffic remains untagged (or tagged on data VLAN). |

**Benefits:**
- Prioritization (QoS) can be applied to voice VLAN for consistent call quality.
- Data traffic (bursty, high-volume) does not interfere with voice traffic.
- Single cable supports both devices.

---

#### Legacy Protocol: ISL

| Protocol | Description |
| :--- | :--- |
| **ISL (Inter-Switch Link)** | Cisco-proprietary VLAN tagging protocol. Predates 802.1Q. Encapsulates the entire frame rather than inserting a tag. |
| **Status** | Obsolete. Modern networks use **802.1Q** (the open standard). |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **VLANs Provide Logical Segmentation** | They separate broadcast domains on a single physical switch, improving security, performance, and management. |
| **802.1Q Trunking Enables VLANs Across Switches** | A single trunk link carries traffic for all VLANs by adding a VLAN tag to each frame. |
| **Native VLAN is Untagged** | On a trunk, the native VLAN is the only one whose traffic is not tagged; it must match on both ends. |
| **Layer 3 Switches Route Between VLANs** | Using SVIs, a layer 3 switch can perform inter-VLAN routing without an external router. |
| **Voice VLANs Separate Voice and Data** | Ensures call quality by isolating voice traffic from noisy data traffic on the same physical cable. |
| **ISL is Obsolete** | 802.1Q is the universal standard for VLAN tagging. |
