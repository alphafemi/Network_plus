

### Static Routing Cheat Sheet

This guide covers the basics of how routers forward traffic, the importance of routing tables, and the configuration and trade-offs of static routing.

---

#### How a Router Forwards Traffic (Simplified)

When a packet arrives at a router, the router follows a simple step-by-step process:

| Step | Action |
| :--- | :--- |
| **1. Identify Destination** | Examine the packet's destination IP address. |
| **2. Consult Routing Table** | Look up the destination in the routing table to find the best path. |
| **3. Forward or Drop** | - If a matching route exists, forward the packet out the appropriate interface (to a directly connected subnet or to a next-hop router). <br> - If no matching route exists, **discard the packet**. |

---

#### Directly Connected Routes

A router automatically knows about networks that are **directly connected** to its interfaces. These are added to the routing table automatically when an interface is configured with an IP address and subnet mask.

**Example: Router 1's Directly Connected Networks**
```
Router 1
  ├── Interface A: 10.10.10.0/24  (connected to Sam's network)
  ├── Interface B: 10.10.40.0/24  (connected to Router 2)
  └── Interface C: 10.10.50.0/24  (connected to Router 3)
```

**Routing Table (before static routes):**
| Destination Network | Next Hop / Interface |
| :--- | :--- |
| `10.10.10.0/24` | Connected (Interface A) |
| `10.10.40.0/24` | Connected (Interface B) |
| `10.10.50.0/24` | Connected (Interface C) |

**The Problem:** If Sam (`10.10.10.2`) tries to send traffic to Jack (`10.10.20.2`), Router 1 does **not** have an entry for `10.10.20.0/24` in its routing table. The packet is **discarded**.

---

#### Static Routing

**Static routing** is the manual configuration of routes in a router's routing table by an administrator. These routes tell the router where to forward traffic for networks that are not directly connected.

| Aspect | Description |
| :--- | :--- |
| **Configuration** | Administrator manually enters each route on each router. |
| **Use Case** | Best for small networks or **stub networks** (networks with a single connection to the rest of the network). |
| **Control** | Complete control over path selection. |

---

#### Configuring Static Routes (Example)

To enable Router 1 to reach networks beyond Router 2 and Router 3, the administrator adds static routes:

| Target Network | Next Hop (Gateway) | Description |
| :--- | :--- | :--- |
| `10.10.20.0/24` | `10.10.40.2` | Send traffic for Jack's network to Router 2 |
| `10.10.30.0/24` | `10.10.50.2` | Send traffic for the other network to Router 3 |

**Updated Routing Table (after static routes):**
| Destination Network | Next Hop / Interface |
| :--- | :--- |
| `10.10.10.0/24` | Connected (Interface A) |
| `10.10.40.0/24` | Connected (Interface B) |
| `10.10.50.0/24` | Connected (Interface C) |
| `10.10.20.0/24` | **Next-hop: 10.10.40.2** (Router 2) |
| `10.10.30.0/24` | **Next-hop: 10.10.50.2** (Router 3) |

Now, when Sam sends traffic to Jack (`10.10.20.2`):
1. Router 1 receives the packet.
2. Router 1 looks up `10.10.20.2` → matches `10.10.20.0/24`.
3. Router 1 forwards the packet to `10.10.40.2` (Router 2).
4. Router 2 has `10.10.20.0/24` as a directly connected network and delivers the packet to Jack.

#### Advantages of Static Routing

| Advantage | Description |
| :--- | :--- |
| **Simplicity** | Easy to configure for small networks. |
| **No Overhead** | No CPU cycles or bandwidth consumed by routing protocol updates. |
| **Security** | Routes are not advertised; no risk of route spoofing or unauthorized route updates. |
| **Predictability** | Traffic takes exactly the path the administrator defines. |

---

#### Disadvantages of Static Routing

| Disadvantage | Description |
| :--- | :--- |
| **Not Scalable** | Manually configuring hundreds or thousands of routers is impractical and error-prone. |
| **No Automatic Failover** | If a link fails, traffic is black-holed until an administrator manually changes the route. |
| **No Adaptation** | Routes do not change in response to network topology changes. |
| **Risk of Routing Loops** | Misconfiguration (e.g., Router A points to Router B, and Router B points back to Router A) can create loops. |

---

#### When to Use Static Routing

| Scenario | Reason |
| :--- | :--- |
| **Stub Networks** | A small remote office with a single WAN link only needs a default route. |
| **Very Small Networks** | A network with only 2–5 routers may be easier to manage with static routes than with a dynamic protocol. |
| **Security-Sensitive Segments** | Where you want absolute control over path selection and no route advertisements. |
| **Default Routes** | A "gateway of last resort" (default route) is often configured statically even when dynamic routing is used. |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Routers Need Routes** | A router only knows about directly connected networks. All other networks must be learned via static or dynamic routes. |
| **Static Routes Are Manual** | Administrators must configure them on each router. |
| **Static Routing Is Simple but Not Scalable** | Works well for small networks but becomes unmanageable at scale. |
| **No Redundancy** | Static routes cannot automatically fail over to backup links. |
| **Stub Networks Are Ideal Candidates** | A single exit point makes static routing a perfect fit. |
