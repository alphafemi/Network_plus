This  provides a clear and practical overview of several network attacks: **rogue DHCP servers**, **rogue access points** (including **evil twins**), and **on‑path attacks** (formerly called man‑in‑the‑middle). It also explains the mitigations available for each.



---

### Rogue Devices and On‑Path Attacks Cheat Sheet

This guide covers how attackers use rogue infrastructure devices to intercept traffic, and the defenses that prevent these attacks.

---

#### Rogue DHCP Server

| Aspect | Description |
| :--- | :--- |
| **What It Is** | An unauthorized DHCP server placed on the network by an attacker (or an unaware employee). |
| **How It Works** | DHCP has no built‑in authentication. A rogue server can respond to DHCP requests faster than the legitimate server. |
| **Impact** | Hands out incorrect IP configurations (wrong gateway, wrong DNS, duplicate IPs), causing denial of service or redirecting traffic to an attacker. |

**Common Results of a Rogue DHCP Server:**
- Duplicate IP addresses (network disruption)
- Invalid default gateway (no internet access)
- Malicious DNS server (redirects to phishing sites)
- Attacker as the default gateway (on‑path attack)

**Mitigations:**

| Mitigation | Description |
| :--- | :--- |
| **DHCP Snooping** | Enterprise switch feature that examines DHCP messages. Only allows DHCP replies from trusted ports (where the legitimate DHCP server is connected). |
| **DHCP Authorization (Microsoft)** | In Active Directory, you can authorize specific DHCP servers. Unauthorized servers will not start the DHCP service. |
| **802.1X (Network Access Control)** | Requires authentication before a device can access the network, preventing unauthorized devices from being plugged in. |

**If a Rogue DHCP Server is Found:**
1. Remove the rogue server from the network.
2. Force all clients to renew their DHCP leases (to obtain legitimate addresses).

#### Rogue Access Point (AP)

| Aspect | Description |
| :--- | :--- |
| **What It Is** | An unauthorized wireless access point connected to the corporate network (physically or via software). |
| **How It Appears** | An employee trying to improve Wi‑Fi coverage (often not malicious), or an attacker deliberately installing it. |
| **Risk** | Bypasses perimeter security. Anyone within range of the rogue AP can connect to the internal network. |

**Software Rogue AP:** Some operating systems allow wireless sharing, turning a laptop into an AP. This is a rogue AP without any extra hardware.

**Detection Methods:**
- Periodic physical walkthroughs with a wireless analyzer.
- Regular network scans for unauthorized APs.
- Wireless intrusion detection systems (WIDS).

**Mitigations:**

| Mitigation | Description |
| :--- | :--- |
| **802.1X (NAC)** | Devices must authenticate before gaining network access. A rogue AP cannot obtain an IP address or reach the network. |
| **Wireless Intrusion Prevention System (WIPS)** | Actively monitors for rogue APs and can block them (e.g., by sending deauthentication frames). |
| **Physical Security** | Lock wiring closets; secure unused Ethernet jacks. |

---

#### Evil Twin (Malicious Rogue AP)

| Aspect | Description |
| :--- | :--- |
| **What It Is** | A rogue access point intentionally configured to impersonate a legitimate Wi‑Fi network. |
| **SSID** | Same or very similar name as the legitimate network (e.g., "CorpWiFi" vs. "CorpWiFi_Free"). |
| **Security Settings** | May mimic the same encryption type (WPA2, WPA3) or present an identical captive portal. |
| **Power** | Often uses higher transmit power to overpower the legitimate AP, ensuring clients connect to the evil twin instead. |

**Attack Flow:**
1. Attacker sets up an evil twin with the same SSID as the corporate Wi‑Fi.
2. Victims connect automatically (devices prefer stronger signal).
3. Attacker captures all traffic (including unencrypted passwords) or performs an on‑path attack.

**Prevention:**
- **Always use encrypted traffic:** VPN, HTTPS, DoH/DoT. Even if connected to an evil twin, encryption protects the data.
- **Use 802.1X with certificate validation:** Clients can verify the authenticity of the AP.
- **WIPS:** Can detect and block evil twins.

---

#### On‑Path Attack (formerly Man‑in‑the‑Middle / MITM)

| Aspect | Description |
| :--- | :--- |
| **Definition** | An attacker positions themselves between two communicating parties, intercepting, monitoring, and optionally modifying traffic. |
| **Why "On‑Path"** | The term "man‑in‑the‑middle" is being replaced because the attacker sits on the communication path, not necessarily "in the middle" geographically. |

**Examples of On‑Path Attacks:**
- Evil twin wireless AP
- ARP poisoning (ARP spoofing)
- Session hijacking
- HTTPS spoofing (SSL stripping)
- DNS poisoning (when combined with interception)

**Key Characteristic:** Neither the source nor the destination device knows the attacker is present. The attacker forwards traffic so the connection appears normal.

**Defense:**

| Defense | How It Works |
| :--- | :--- |
| **Encryption (TLS, VPN, SSH)** | Even if an attacker intercepts the traffic, they cannot read or modify it without breaking encryption. |
| **Certificate Validation** | Clients must validate server certificates; do not bypass certificate warnings. |
| **HSTS (HTTP Strict Transport Security)** | Forces browsers to always use HTTPS for a domain, preventing SSL stripping. |
| **ARP Spoofing Defenses** | Dynamic ARP Inspection (DAI), static ARP entries. |

---

#### Comparison of Attacks

| Attack | Target | Method | Primary Defense |
| :--- | :--- | :--- | :--- |
| **Rogue DHCP Server** | DHCP clients | Unauthorized DHCP responses | DHCP snooping, 802.1X |
| **Rogue AP (non‑malicious)** | Wireless clients | Unauthorized AP on network | 802.1X, WIPS, physical security |
| **Evil Twin (malicious rogue AP)** | Wireless clients | Impersonated SSID | Encryption (VPN/HTTPS), certificate validation, WIPS |
| **On‑Path (general)** | Any communication | Intercepting traffic between two parties | Encryption (end‑to‑end) |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **DHCP has no authentication** | A rogue DHCP server can disrupt or redirect your network. |
| **DHCP snooping prevents rogue servers** | Configure trusted ports on switches where legitimate DHCP servers are connected. |
| **Rogue APs bypass perimeter security** | Anyone in range can connect to your internal network. |
| **802.1X (NAC) prevents unauthorized devices** | Devices must authenticate before gaining network access. |
| **Evil twins impersonate legitimate Wi‑Fi** | Use higher power to attract victims; always use encryption to protect your data. |
| **On‑path attacks intercept communications** | Encryption (VPN, HTTPS, SSH) renders intercepted data useless to the attacker. |
| **WIPS can detect and block rogue APs** | Active monitoring for unauthorized wireless devices. |
