This  provides a practical overview of **device security** best practices. It covers controlling network access via port management, securing switch ports, managing authentication, and centralizing cryptographic key management.



---

### Device Security Cheat Sheet

This guide covers network‑level device hardening, port security, access control, and key management.

---

#### Controlling Open Ports (Service Exposure)

| Aspect | Description |
| :--- | :--- |
| **The Risk** | Every network service listening on a port (0–65535) is a potential entry point into your system. |
| **Best Practice** | Only keep ports open for services that are actively in use. Disable or remove unused services. |
| **Detection Tool** | Use **Nmap** (or similar port scanner) to discover which ports are open on your system. |
| **Control** | Use a **firewall** (host‑based or network) to restrict which source IPs can access open ports (e.g., allow only internal subnet). |

**Why It Matters:** An open port on a service you forgot about (e.g., a development web server, an old database) can be exploited by an attacker who scans your network.

---

#### Default Credentials

| Aspect | Description |
| :--- | :--- |
| **The Problem** | Many switches, routers, firewalls, and IoT devices ship with default usernames and passwords (e.g., `admin/admin`, `root/blank`). |
| **Risk** | If not changed, anyone who knows the default credentials (easily found online) can gain administrative access to the device. |
| **Database** | `routerpasswords.com` maintains a large database of default credentials for many devices. |
| **Best Practice** | Always change default credentials during initial setup. Use strong, unique passwords. |

---

#### Switch Port Security

Port security prevents unauthorized devices from connecting to a switch port.

| Feature | Description |
| :--- | :--- |
| **How It Works** | The switch learns the MAC address(es) of device(s) connected to a port and only allows traffic from those addresses. |
| **Configuration Options** | - **Maximum MAC count:** How many different MAC addresses are allowed on this port (e.g., 1 for a single workstation, 5 for a printer/phone). <br> - **Sticky MAC:** Learn and save the first MAC address seen. <br> - **Violation action:** What to do when an unauthorized device connects (e.g., shutdown the port, restrict, protect). |
| **Default Violation Action (many switches)** | Shut down the port and send an alert to the administrator. |

**Example (Cisco‑like):**
```
interface GigabitEthernet0/1
 switchport mode access
 switchport port-security
 switchport port-security maximum 1
 switchport port-security violation shutdown
 switchport port-security mac-address sticky
```

**What It Prevents:** Someone unplugging a legitimate device and plugging in their own laptop (MAC address mismatch → port shuts down).

#### Disable Unused Interfaces

| Aspect | Description |
| :--- | :--- |
| **The Risk** | Unused network jacks (in conference rooms, empty cubicles, break rooms) are physical entry points. |
| **Best Practice** | Administratively disable (shut down) any switch port that is not actively in use. |
| **Operational Cost** | Requires tracking which ports are enabled/disabled and re‑enabling them when needed (more administration, but more secure). |

---

#### Network Access Control (NAC) / 802.1X

| Aspect | Description |
| :--- | :--- |
| **Definition** | A device must authenticate before being granted network access (wired or wireless). |
| **Standard** | **802.1X** (IEEE standard for port‑based network access control). |
| **How It Works** | When you plug in (or connect to Wi‑Fi), the switch/AP prompts for authentication (username/password, certificate). Only after successful authentication does the device get an IP address and network access. |
| **Benefit** | Even if an attacker plugs into a live wall jack, they cannot access the network without valid credentials. |

---

#### MAC Address Filtering (Weak Control)

| Aspect | Description |
| :--- | :--- |
| **How It Works** | The network allows only specific MAC addresses (hardware addresses) to communicate. |
| **Problem** | MAC addresses can be **spoofed** (changed administratively in software). An attacker can capture a valid MAC address from the network and then configure their device to use that address. |
| **Classification** | **Security through obscurity** – it adds a layer, but it is easily bypassed by a knowledgeable attacker. |
| **Best Use** | As one layer in a defense‑in‑depth strategy, not as a primary control. |

---

#### Key Management

| Aspect | Description |
| :--- | :--- |
| **The Challenge** | Organizations have many keys: SSL/TLS certificates (web servers), SSH keys (Linux servers, network devices), API keys (cloud services), encryption keys. |
| **Key Management System (KMS)** | Centralized platform to create, store, renew, revoke, and monitor all cryptographic keys and certificates. |
| **Capabilities** | - List all certificates and keys <br> - Track expiration dates <br> - Automate renewal <br> - See which users/devices are using which keys <br> - Generate reports (SSH key usage, certificate authorities, vulnerabilities) |

**Why Centralized Key Management Matters:**
- Prevents expired certificates from causing outages.
- Prevents orphaned SSH keys (former employees still have access).
- Provides audit trail for compliance (PCI DSS, HIPAA, SOC2).

#### Summary Table: Device Security Controls

| Control | What It Does | Strength | Best Practice |
| :--- | :--- | :--- | :--- |
| **Firewall (port filtering)** | Restricts which IPs can access open ports | Strong | Default deny; allow only necessary services |
| **Change default credentials** | Removes well‑known backdoors | Strong | Always do this on new devices |
| **Port security (MAC limiting)** | Prevents unauthorized devices on a switch port | Strong (for physical access) | Use on access ports |
| **Disable unused interfaces** | Removes physical entry points | Strong | Shut down all unused switch ports |
| **802.1X (NAC)** | Requires authentication before network access | Very strong | Implement for wired and wireless |
| **MAC address filtering** | Allows only specific MAC addresses | Weak (easily spoofed) | Use only as an additional layer |
| **Key Management System** | Centralized control of certificates, SSH keys, API keys | Essential for scale | Use for any environment with >10 servers |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Scan your open ports** | Use Nmap to discover unexpected services; close unused ports with a firewall. |
| **Change default credentials immediately** | Default passwords are publicly documented. |
| **Switch port security prevents device swapping** | Configure maximum MAC addresses and violation actions. |
| **Disable unused switch ports** | Every live jack is a potential entry point. |
| **802.1X (NAC) is strong access control** | Authentication required before network access. |
| **MAC filtering is weak** | Easily bypassed by spoofing; use only as an extra layer. |
| **Centralized key management is essential at scale** | Prevents expired certificates, orphaned keys, and compliance failures. |
