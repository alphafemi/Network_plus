This  provides a broad overview of essential IT security concepts, from data classification and encryption to identity management, access control, and physical security. It covers the key areas that network professionals need to understand to protect data and systems effectively.



---

### IT Security Concepts Cheat Sheet

This guide covers data states, encryption, trust models, identity and access management, and physical security controls.

---

#### Data States (Where Protection Is Applied)

| Data State | Also Known As | Description | Protection Methods |
| :--- | :--- | :--- | :--- |
| **Data in Transit** | Data in Motion | Data actively moving across a network (wired or wireless). | TLS, IPsec, VPNs |
| **Data at Rest** | Stored Data | Data stored on hard drives, SSDs, databases, backups, or cloud storage. | Full disk encryption, database encryption, file/folder encryption, ACLs |

---

#### Trust and Certificate Management

| Concept | Description | Example |
| :--- | :--- | :--- |
| **PKI (Public Key Infrastructure)** | The policies, processes, and systems for managing digital certificates and encryption keys. | Managing web server certificates, user certificates, device certificates. |
| **Certificate Authority (CA)** | A trusted entity that issues and digitally signs digital certificates. | DigiCert (public CA), internal Windows CA (private CA). |
| **Web of Trust** | A distributed trust model where users sign each other's certificates (no central CA). | PGP (Pretty Good Privacy) for email encryption. |

**How Trust Works with a CA:**
1. A CA signs a certificate.
2. Anyone who trusts the CA automatically trusts any certificate signed by that CA.
3. Public CAs are trusted by default in most operating systems and browsers.
4. Private (self‑signed) CAs can be used inside an organization; all internal devices must be configured to trust that CA.

#### Identity and Access Management (IAM)

IAM is the framework of policies and technologies that ensure the right people have the right access to the right resources.

| IAM Component | Description |
| :--- | :--- |
| **Authentication** | Verifying a user's identity (e.g., username/password, MFA). |
| **Authorization** | Determining what an authenticated user is allowed to do (e.g., read, write, delete). |
| **Accounting (Auditing)** | Tracking and logging user access for compliance and investigation. |

---

#### Least Privilege Principle

| Principle | Description | Why It Matters |
| :--- | :--- | :--- |
| **Least Privilege** | Users (and applications) are granted only the minimum rights and permissions necessary to perform their job functions. | Reduces the impact of compromised credentials; prevents accidental or malicious misuse of sensitive data. |

**Example:** A help desk technician should have read‑only access to the hardware database, while the help desk manager may have write access. An accounting user should have no access to the hardware database.

---

#### Access Control Models

| Model | Description | Implementation |
| :--- | :--- | :--- |
| **Role‑Based Access Control (RBAC)** | Permissions are assigned to **roles**, and users are assigned to roles based on their job function. | Windows security groups (e.g., "ShippingReceiving" group, "Managers" group). |
| **Geographic Restriction** | Access is allowed or denied based on the user's physical location. | Allow access only from corporate headquarters IP range; restrict access from other countries. |

**Geographic Methods:**
- **IP geolocation** – Rough location based on IP address.
- **GPS** – Precise location from a mobile device.
- **WiFi network name** – Location inferred from the access point.

**Geofencing:** Creating a virtual geographic boundary. Access is granted only when the user is physically inside that boundary.

---

#### Physical Security Controls

| Control | Description | Examples |
| :--- | :--- | :--- |
| **CCTV (Closed Circuit Television)** | Video surveillance cameras networked to a central recording system. | Motion detection, license plate recognition, facial recognition, long‑term video storage. |
| **Door Locks** | Physical access control to rooms, buildings, or data center cages. | Traditional key locks, electronic keypads (PIN), RFID badge readers, biometric readers (fingerprint, retina, hand print). |
| **Multi‑Factor Physical Access** | Combining two or more factors for physical entry. | Badge + PIN (prevents use of a lost badge alone). |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Data in transit needs encryption** | TLS and IPsec protect data as it crosses the network. |
| **Data at rest needs encryption + ACLs** | Full disk encryption, file encryption, and access control lists protect stored data. |
| **PKI manages trust** | Certificates and CAs establish trust for encryption, authentication, and code signing. |
| **Least privilege is a core security principle** | Limit access to only what is necessary for a job function. |
| **RBAC simplifies permission management** | Assign permissions to roles, not individual users. |
| **Physical security is part of overall security** | Cameras, locks, and access control protect the hardware that stores and processes data. |
