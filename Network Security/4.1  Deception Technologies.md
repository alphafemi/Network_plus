### Security Concepts: Honeypots, Risk, Vulnerabilities, and the CIA Triad Cheat Sheet

This guide covers deception technologies, risk terminology, and the three core principles of information security.

---

#### Deception Technologies: Honeypots and Honeynets

These are active defense mechanisms designed to attract attackers and study their behavior.

| Concept | Description | Key Characteristics |
| :--- | :--- | :--- |
| **Honeypot** | A single decoy system (server, workstation, or service) designed to look attractive to attackers. | - Often virtual <br> - Contains fake "honey" data <br> - Used to observe attack techniques |
| **Honeynet** | A network of multiple decoy systems (servers, workstations, routers, firewalls, NAS, etc.) that together form a realistic but fake environment. | - Larger deception framework <br> - More convincing than a single honeypot <br> - Can include many virtual components |

**Why Use Honeypots/Honeynets?**
- Attract automated attack scripts (most attacks are not human‑typed in real time).
- Observe attacker techniques and tools without exposing real production systems.
- Learn about new vulnerabilities and exploits before they are widely used.
- Can feed threat intelligence into other security systems.

**Resource:** `projecthoneypot.org` – a good starting point for learning about and building honeypots.

#### Risk, Vulnerabilities, Exploits, and Threats

These terms are often used together but have distinct meanings.

| Term | Definition | Example |
| :--- | :--- | :--- |
| **Risk** | The potential exposure to harm or loss. The likelihood that a threat will exploit a vulnerability. | A company decides to add a new public‑facing web application; there is a risk that it could be hacked. |
| **Vulnerability** | A weakness in a system, application, or process that could be exploited. | An unpatched security flaw in Windows, a misconfigured firewall rule, weak authentication. |
| **Exploit** | The method or code used to take advantage of a vulnerability. | A specific piece of malware that uses a buffer overflow to gain administrative access. |
| **Threat** | The potential cause of an unwanted incident (a person, event, or natural disaster). | An external attacker, a disgruntled employee, a fire, a flood, a lightning strike. |
| **Threat Agent** | The entity that performs the threat action (often a person or automated script). | A hacker running a scanning tool; a malware botnet. |
| **Threat Action** | The specific step taken to exploit a vulnerability. | Sending a crafted packet, injecting SQL code, uploading a webshell. |

**The Chain:**
```
Vulnerability (weakness) + Threat Agent (attacker) + Threat Action (exploit) = Security Incident
```

#### The CIA Triad (Fundamental Principles of IT Security)

The CIA triad (sometimes called the AIC triad to avoid confusion with the US Central Intelligence Agency) consists of three core principles.

| Principle | Description | Protection Methods |
| :--- | :--- | :--- |
| **Confidentiality** | Ensuring that data is only accessible to authorized individuals. | Encryption, access control lists (ACLs), file permissions, multi‑factor authentication. |
| **Integrity** | Ensuring that data has not been tampered with or modified by unauthorized parties. | Digital signatures, hashing (e.g., SHA‑256), checksums, version control, write‑once media. |
| **Availability** | Ensuring that systems and data are accessible to authorized users when needed. | Redundancy (RAID, failover clusters), backups, disaster recovery plans, DDoS protection. |

**Real‑World Example (Email):**
- **Confidentiality:** The email is encrypted (TLS) so no one can read it in transit.
- **Integrity:** The email is digitally signed so the recipient can verify it wasn't altered.
- **Availability:** The email server is redundant and backed up so users can always access their mail.

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Honeypots attract attackers** | Decoy systems that look real to catch automated attacks and study attacker behavior. |
| **Honeynets are larger decoy networks** | Multiple fake systems (routers, servers, workstations) create a convincing deception environment. |
| **Vulnerability is a weakness** | A flaw in software, hardware, or configuration that could be exploited. |
| **Exploit is the attack method** | The specific technique or code that takes advantage of a vulnerability. |
| **Threat is the potential cause** | An attacker, a script, a fire, or a flood that could cause harm. |
| **Risk is the likelihood of harm** | The combination of a threat exploiting a vulnerability, and the potential impact. |
| **CIA triad = Confidentiality, Integrity, Availability** | The three core principles that every security control should support. |
