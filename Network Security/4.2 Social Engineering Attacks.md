This  provides a clear and practical overview of **social engineering** techniques, with a focus on **phishing**, **shoulder surfing**, **tailgating/piggybacking**, and **dumpster diving**. It explains how attackers exploit human psychology and physical security weaknesses, along with practical defenses.



---

### Social Engineering Attacks Cheat Sheet

This guide covers common social engineering techniques and how to defend against them.

---

#### What is Social Engineering?

| Aspect | Description |
| :--- | :--- |
| **Definition** | Manipulating people into divulging confidential information or performing actions that compromise security. |
| **Key Ingredients** | Spoofing (pretending to be someone else) + psychological manipulation. |
| **Target** | Human weakness (trust, helpfulness, fear, urgency) rather than technical vulnerabilities. |

---

#### Phishing (Email‑Based)

| Aspect | Description |
| :--- | :--- |
| **How It Works** | Attacker sends an email that appears to come from a legitimate source (bank, IT support, cloud provider, etc.) to trick the victim into clicking a malicious link or providing credentials. |
| **Common Red Flags** | Sender email address doesn't match the claimed domain; poor grammar or spelling; mismatched fonts; generic greetings ("Dear User"); urgent or threatening language. |

**Example (from the video):**
- **Claimed sender:** "Rackspace Service"
- **Actual sender:** `@icloud.com` (not the Rackspace domain)
- **Message:** "Your email has not been confirmed for the new upgraded service"
- **Fake login page:** Looks similar to the real Rackspace login page but with subtle graphic differences.

**Defenses:**

| Defense | Description |
| :--- | :--- |
| **Never click links in emails** | Navigate to the site manually by typing the URL into your browser. |
| **Check the sender's email address** | Hover over the "From" field to see the actual address. |
| **Look for red flags** | Grammar, spelling, generic greetings, urgency. |
| **Use a virtual machine** | If you must test a suspicious link, use an isolated VM (as the instructor did). |
| **Email filtering** | SPF, DKIM, DMARC help prevent spoofed emails from reaching inboxes. |
| **Security awareness training** | Regular phishing simulations and education. |

#### Shoulder Surfing

| Aspect | Description |
| :--- | :--- |
| **Definition** | Watching someone's screen or keyboard to capture sensitive information (passwords, confidential data, personal information). |
| **Locations** | Airports, coffee shops, airplanes, open office plans, public transportation. |
| **Advanced Variants** | Using binoculars or telescopes from a distance; using malware to enable a device's camera. |

**Defenses:**

| Defense | Description |
| :--- | :--- |
| **Privacy filters** | LCD filters that make the screen visible only from directly in front (side views appear black). |
| **Be aware of your surroundings** | Sit with your back to a wall; avoid sensitive work in public spaces. |
| **Position screens away from windows** | Prevents viewing from outside and reduces glare. |
| **Lock your screen** | When stepping away, even briefly. |
| **Camera covers** | Physical covers for laptop cameras to prevent remote viewing. |

#### Physical Access Attacks: Tailgating and Piggybacking

Both techniques involve gaining unauthorized entry to a secured building or area.

| Term | Description | Victim's Awareness |
| :--- | :--- | :--- |
| **Tailgating** | Following an authorized person through a door without their knowledge (e.g., catching the door before it closes). | Unaware |
| **Piggybacking** | An authorized person knowingly holds the door for an unauthorized person (often out of politeness). | Aware (and complicit) |

**Example:** Someone carrying boxes or donuts asks an employee to hold the door. The employee helps, allowing the attacker inside.

**Defenses:**

| Defense | Description |
| :--- | :--- |
| **Security awareness training** | Employees should challenge people without badges. |
| **Visitor badges** | Clearly visible badges for all guests. |
| **One person, one badge** | Each person must badge in individually; doors close fully between entries. |
| **Access control vestibule (airlock)** | A small room with two doors. Only one person can be inside at a time; the first door must close before the second opens. |
| **Report unattended strangers** | Ask, "May I help you?" or call security if someone is without a badge. |

#### Dumpster Diving

| Aspect | Description |
| :--- | :--- |
| **Definition** | Going through an organization's trash to find sensitive information (printed documents, old hard drives, sticky notes, etc.). |
| **What Attackers Find** | Names, email addresses, phone numbers, organizational charts, passwords on sticky notes, financial information, network diagrams. |
| **Legality (US)** | Generally legal if the trash is in a public place (curbside, public alley). Illegal if the bin is on private property with "No Trespassing" signs or if local/state laws prohibit it. |

**Defenses:**

| Defense | Description |
| :--- | :--- |
| **Lock dumpsters** | Place bins in a fenced, locked area with access control. |
| **Shred sensitive documents** | Cross‑cut shredding (or pulping) for paper. |
| **Third‑party shredding services** | Mobile shredding trucks that come to your location. |
| **Burn (incineration)** | Used by governments for highly classified material. |
| **Audit your own trash** | Look through your garbage to see what an attacker would find. |

---

#### Common Themes Across Social Engineering Attacks

| Theme | Description |
| :--- | :--- |
| **Exploit trust** | Attackers rely on people's natural inclination to be helpful or trusting. |
| **Physical security matters** | Many breaches start with physical access (tailgating, dumpster diving). |
| **Training is the primary defense** | Technical controls help, but employee awareness is critical. |
| **Layered security** | No single control is sufficient; combine policies, physical security, and technology. |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Phishing exploits trust in email** | Always verify the sender and never click links in unexpected emails. |
| **Shoulder surfing is common in public** | Use privacy filters and be aware of your surroundings. |
| **Tailgating and piggybacking bypass physical security** | Challenge people without badges; use airlocks or one‑badge‑per‑entry policies. |
| **Dumpster diving recovers sensitive information** | Shred documents and secure your trash bins. |
| **Social engineering targets humans, not systems** | Technology cannot fully prevent these attacks; awareness training is essential. |
