This  introduces the **Seven-Second Subnetting** method, a streamlined shortcut that builds on the Magic Number method but eliminates the need for any calculation beyond simple addition and subtraction of 1. By using a pre-drawn chart, you can quickly determine network addresses, broadcast addresses, and usable host ranges without any math.



---

### The Seven-Second Subnetting Method Cheat Sheet

This method relies on a single reference chart that you create at the start of your exam. Once the chart is drawn, the entire subnetting process becomes a matter of looking up values and following simple rules.

#### Step 2: The Four-Step Process

Once your chart is ready, follow these steps for any subnetting problem.

| Step | Action |
| :--- | :--- |
| **1. Convert CIDR to Decimal** | Use the chart to find the decimal subnet mask that matches the given CIDR notation. |
| **2. Determine the Subnet (Network) Address** | - If the mask octet = **255**: Copy the IP address octet down. <br> - If the mask octet = **0**: Write **0** in that octet. <br> - If the mask octet is **other** (128, 192, 224, etc.): Find the block that contains the IP's value in the **Addresses per Subnet** chart and use the **start** of that block. |
| **3. Determine the Broadcast Address** | - If the mask octet = **255**: Copy the IP address octet down. <br> - If the mask octet = **0**: Write **255** in that octet. <br> - If the mask octet is **other**: Find the same block as above, and use the **last address in that block** (start of next block - 1). |
| **4. Calculate Usable Hosts** | - **First Usable:** Network Address + 1 <br> - **Last Usable:** Broadcast Address - 1 |

#### Worked Examples

**Example 1: 165.245.12.88/24**

| Step | Calculation |
| :--- | :--- |
| **1. Decimal Mask** | /24 = `255.255.255.0` |
| **2. Network Address** | Octets 1-3 (mask 255) → copy IP: `165.245.12` <br> Octet 4 (mask 0) → 0 <br> **Network = 165.245.12.0** |
| **3. Broadcast Address** | Octets 1-3 (mask 255) → copy IP: `165.245.12` <br> Octet 4 (mask 0) → 255 <br> **Broadcast = 165.245.12.255** |
| **4. Usable Hosts** | First: 165.245.12.1 <br> Last: 165.245.12.254 |

**Example 2: 165.245.12.88/26**

| Step | Calculation |
| :--- | :--- |
| **1. Decimal Mask** | /26 = `255.255.255.192` (Addresses per subnet = 64) |
| **2. Network Address** | Octets 1-3 (mask 255) → copy IP: `165.245.12` <br> Octet 4 (mask 192) → 88 is in the 64-127 block → **64** <br> **Network = 165.245.12.64** |
| **3. Broadcast Address** | Octets 1-3 (mask 255) → copy IP: `165.245.12` <br> Octet 4 (mask 192) → last address in 64-127 block → **127** <br> **Broadcast = 165.245.12.127** |
| **4. Usable Hosts** | First: 165.245.12.65 <br> Last: 165.245.12.126 |

**Example 3: 18.172.200.77/17**

| Step | Calculation |
| :--- | :--- |
| **1. Decimal Mask** | /17 = `255.255.128.0` (Addresses per subnet = 32,768) |
| **2. Network Address** | Octet 1 (mask 255) → copy IP: `18` <br> Octet 2 (mask 255) → copy IP: `172` <br> Octet 3 (mask 128) → 200 is in the 128-255 block → **128** <br> Octet 4 (mask 0) → 0 <br> **Network = 18.172.128.0** |
| **3. Broadcast Address** | Octet 1 (mask 255) → copy IP: `18` <br> Octet 2 (mask 255) → copy IP: `172` <br> Octet 3 (mask 128) → last address in 128-255 block → **255** <br> Octet 4 (mask 0) → 255 <br> **Broadcast = 18.172.255.255** |
| **4. Usable Hosts** | First: 18.172.128.1 <br> Last: 18.172.255.254 |

#### Exam Tips

| Tip | Description |
| :--- | :--- |
| **Practice Drawing the Chart** | Before the exam, practice recreating the chart from memory until you can do it quickly and accurately. |
| **Use the Whiteboard** | In a testing center, use the provided dry-erase board. If the pens are thick, consider bringing your own fine-tip dry-erase pen (check with the center first). |
| **Find Your Method** | The Seven-Second method is one of many. Whether you use this, the Magic Number method, or another approach, choose the one that feels most intuitive and reliable for you. |
