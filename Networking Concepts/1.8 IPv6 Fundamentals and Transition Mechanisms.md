### IPv6 Fundamentals and Transition Mechanisms Cheat Sheet

This guide covers the structure of IPv6 addresses, compression rules, and the methods used to bridge IPv4 and IPv6 networks.

---

#### Why IPv6?

| Problem | IPv4 | IPv6 Solution |
| :--- | :--- | :--- |
| **Address Exhaustion** | 32-bit addresses → ~4.29 billion total addresses. All IPv4 address blocks are exhausted. | 128-bit addresses → 340 undecillion (3.4 × 10³⁸) addresses. |
| **NAT Dependence** | NAT is required to connect multiple private devices to the internet, causing complexity and breaking some applications. | Enough addresses for every device to have a globally unique, routable address. NAT is no longer required. |
| **Scale** | Cannot support the estimated 20+ billion internet-connected devices. | Can support massive scale (e.g., every grain of sand on Earth could have 45 quintillion addresses). |

---

#### IPv6 Address Structure

An IPv6 address is **128 bits** (16 bytes), typically written as eight groups of **16 bits** (two bytes) in hexadecimal, separated by colons.

| Aspect | Description |
| :--- | :--- |
| **Format** | `hhhh:hhhh:hhhh:hhhh:hhhh:hhhh:hhhh:hhhh` (each `h` = hexadecimal digit 0–f) |
| **Example** | `2600:DDDD:1111:0001:0000:0000:0000:0001` |
| **Binary Representation** | Each 16-bit group corresponds to 4 hexadecimal characters. |

---

#### IPv6 Address Compression Rules

IPv6 addresses can be compressed to make them easier to read and work with.

| Rule | Description | Example |
| :--- | :--- | :--- |
| **1. Remove Leading Zeros** | Leading zeros within each 16-bit group can be omitted. | `0001` → `1` <br> `04C3` → `4C3` |
| **2. Double Colon (::)** | One (and only one) contiguous block of all-zero groups can be replaced with `::`. | `0000:0000:0000:0001` → `::1` |

**Compression Example 1:**

| Full Address | After Removing Leading Zeros | After Double Colon Compression |
| :--- | :--- | :--- |
| `2600:DDDD:1111:0001:0000:0000:0000:0001` | `2600:DDDD:1111:1:0:0:0:1` | `2600:DDDD:1111:1::1` |

**Compression Example 2:**

| Full Address | After Removing Leading Zeros | After Double Colon Compression |
| :--- | :--- | :--- |
| `2601:04C3:4002:BE00:0000:0000:0000:0066` | `2601:4C3:4002:BE00:0:0:0:66` | `2601:4C3:4002:BE00::66` |

#### Transition Mechanisms: Bridging IPv4 and IPv6

IPv4 and IPv6 are not directly compatible. Several methods allow them to coexist and communicate.

| Mechanism | Description | Status / Use Case |
| :--- | :--- | :--- |
| **Tunneling** | Encapsulates one protocol inside the other to traverse networks that don't support it natively. | Historically used; less common today. |
| **Dual Stack** | Devices run both IPv4 and IPv6 simultaneously. Each interface has both an IPv4 and an IPv6 address. Applications can choose which to use. | **Most common modern approach.** Supported by all modern operating systems. |
| **Translation (NAT64/DNS64)** | Translates between IPv4 and IPv6 at the network layer, allowing IPv6-only clients to communicate with IPv4-only servers. | Used when transitioning to IPv6-only networks while maintaining IPv4 access. |

---

#### Tunneling Mechanisms (Legacy)

| Method | Description | Why It's Less Common |
| :--- | :--- | :--- |
| **6to4** | Encapsulates IPv6 traffic inside IPv4 packets. Creates an IPv6 address based on the IPv4 address. Does **not** support NAT. | Requires specialized relay routers; incompatible with NAT; removed from modern Windows versions. |
| **4in6** | Encapsulates IPv4 traffic inside IPv6 packets (the reverse of 6to4). | Uncommon; used only in specific migration scenarios. |

---

#### Dual Stack (Current Standard)

Dual stack is the most common IPv6 deployment method today.

| Feature | Description |
| :--- | :--- |
| **Operation** | A device has both an IPv4 and an IPv6 address on the same interface. |
| **Routing** | Separate routing tables for IPv4 and IPv6. |
| **Application Choice** | Applications can use either protocol based on preference (usually IPv6 preferred if available). |
| **Advantage** | No translation or encapsulation overhead; both protocols work natively. |

---

#### NAT64 and DNS64 (Translation)

NAT64 enables IPv6-only clients to communicate with IPv4-only servers by translating between the two protocols.

| Component | Function |
| :--- | :--- |
| **DNS64** | A DNS server that returns a synthetic IPv6 address for IPv4-only destinations. This address points to the NAT64 router, not the actual destination. |
| **NAT64** | A router that translates between IPv6 and IPv4. It receives IPv6 packets from the client, translates them to IPv4, and forwards them to the IPv4 destination. |

#### IPv6 Address Types (Overview)

| Type | Description | Example |
| :--- | :--- | :--- |
| **Global Unicast** | Publicly routable IPv6 addresses (similar to public IPv4). | `2001:db8::/32` |
| **Link-Local** | Used for communication on a single network segment (not routable). Always starts with `fe80::/10`. | `fe80::5d18:652:6ffd:8f52` |
| **Unique Local** | Private addresses for internal use (similar to RFC 1918). Starts with `fd00::/8`. | `fd00::/8` |
| **Multicast** | One-to-many communication. Starts with `ff00::/8`. | `ff02::1` (all nodes) |
| **Anycast** | One-to-nearest communication (same concept as IPv4 anycast). | (Addresses taken from unicast space) |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **IPv6 Solves Address Exhaustion** | The 128-bit address space is virtually unlimited, eliminating the need for NAT. |
| **IPv6 Addresses Can Be Compressed** | Leading zeros are dropped, and a single contiguous block of zeros can be replaced with `::`. |
| **IPv4 and IPv6 Are Not Compatible** | Direct communication is impossible; transition mechanisms are required. |
| **Dual Stack is the Current Standard** | Most modern networks run both protocols simultaneously, allowing a smooth migration. |
| **NAT64/DNS64 Enables IPv6-Only Networks** | Translation allows IPv6-only clients to reach IPv4-only resources without requiring dual stack on the client. |
| **Tunneling is Legacy** | 6to4 and 4in6 are rarely used today due to complexity and NAT issues. |
