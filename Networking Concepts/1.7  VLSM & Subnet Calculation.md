

### VLSM & Subnet Calculation Cheat Sheet

This guide explains how to determine the number of subnets and hosts per subnet using the powers of 2, based on how many bits are "borrowed" from the host portion of an address.

---

#### The Core Concept: Borrowing Bits

The key to VLSM is the idea of "borrowing" bits from the host portion to extend the network (or subnet) portion.

- **Default Network Bits:** The number of bits defined by the address's original class (e.g., 8 bits for Class A, 16 for Class B, 24 for Class C).
- **Borrowed Bits:** Additional bits taken from the host portion to create more networks.
- **Subnet Bits = Borrowed Bits.**
- **Host Bits = Total bits left after the CIDR mask.**

**Analogy (The Pizza):**
A default network (like a `/8`) is a whole pizza. Using VLSM is like deciding how many slices to cut that pizza into. The more slices (subnets) you cut, the smaller each slice (hosts per subnet) becomes.

---

#### The Calculation Formulas

Using the powers of 2 makes subnet calculation fast and efficient.

| Calculation | Formula | Notes |
| :--- | :--- | :--- |
| **Number of Subnets** | **`2^(Number of Subnet Bits)`** | This tells you how many individual networks you can create. |
| **Number of Hosts per Subnet** | **`2^(Number of Host Bits) - 2`** | The **`-2`** accounts for the **Network Address** (all host bits 0) and the **Broadcast Address** (all host bits 1), which cannot be assigned to devices. |

---

#### Worked Examples

**Example 1: Class A with `/24` (10.1.1.0/24)**

- **Default Mask (Class A):** `/8` (255.0.0.0)
- **New Mask (CIDR):** `/24` (255.255.255.0)

| Component | Bits | Calculation |
| :--- | :--- | :--- |
| **Default Network Bits** | 8 | (The original Class A network) |
| **Borrowed Subnet Bits** | **16** | `/24` - `/8` = 16 bits borrowed from host |
| **Host Bits Remaining** | **8** | 32 total bits - 24 network bits = 8 bits for hosts |
| **Number of Subnets** | 2^16 = **65,536** | A very large number of possible subnets. |
| **Hosts per Subnet** | 2^8 - 2 = 256 - 2 = **254** | Each subnet can have 254 usable devices. |

**Example 2: Class C with `/26` (192.168.11.0/26)**

- **Default Mask (Class C):** `/24` (255.255.255.0)
- **New Mask (CIDR):** `/26` (255.255.255.192)

| Component | Bits | Calculation |
| :--- | :--- | :--- |
| **Default Network Bits** | 24 | (The original Class C network) |
| **Borrowed Subnet Bits** | **2** | `/26` - `/24` = 2 bits borrowed from host |
| **Host Bits Remaining** | **6** | 32 total bits - 26 network bits = 6 bits for hosts |
| **Number of Subnets** | 2^2 = **4** | We can create 4 subnets from this original network. |
| **Hosts per Subnet** | 2^6 - 2 = 64 - 2 = **62** | Each subnet can have 62 usable devices. |

**Example 3: Class B with `/21` (172.16.55.0/21)**

- **Default Mask (Class B):** `/16` (255.255.0.0)
- **New Mask (CIDR):** `/21` (255.255.248.0)

| Component | Bits | Calculation |
| :--- | :--- | :--- |
| **Default Network Bits** | 16 | (The original Class B network) |
| **Borrowed Subnet Bits** | **5** | `/21` - `/16` = 5 bits borrowed from host |
| **Host Bits Remaining** | **11** | 32 total bits - 21 network bits = 11 bits for hosts |
| **Number of Subnets** | 2^5 = **32** | We can create 32 subnets. |
| **Hosts per Subnet** | 2^11 - 2 = 2048 - 2 = **2,046** | Each subnet can have 2,046 usable devices. |

---

#### Key Takeaway: The Trade-off

This process highlights the fundamental trade-off in subnetting:

- **More Subnets** (borrowing more bits) = **Fewer Hosts per Subnet**
- **Fewer Subnets** (borrowing fewer bits) = **More Hosts per Subnet**

VLSM allows you to choose the perfect balance for each segment of your network, optimizing for efficiency and minimizing wasted IP addresses.
