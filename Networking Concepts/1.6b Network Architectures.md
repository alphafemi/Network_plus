### Enterprise Network Architectures Cheat Sheet

This guide covers the common hierarchical network designs and the terminology used to describe data flow within and outside the data center.

---

#### The Three-Tier Architecture

This is the classic, hierarchical model for large enterprise networks. It separates the network into distinct functional layers.

| Layer | Name | Primary Function | Analogy (The City) | Key Characteristics |
| :--- | :--- | :--- | :--- | :--- |
| **Core Layer** | **The Core** | The high-speed backbone of the network. Connects the distribution layer and houses critical shared resources (servers, databases, applications). | **Downtown / City Center:** Where all the major resources (banks, government, main offices) are located. | - Focuses on speed and reliability. <br> - Often uses high-end routers and switches. <br> - Minimal policy enforcement (just fast forwarding). |
| **Distribution Layer** | **The Distribution** | The aggregation point and boundary between the access and core layers. Provides policy-based connectivity. | **The Highways / Main Roads:** Connect your neighborhood to the downtown core. | - Aggregates connections from many access switches. <br> - Enforces policies (QoS, security, routing). <br> - Provides redundancy. |
| **Access Layer** | **The Access** | The point where end-user devices (computers, phones, printers) connect to the network. | **Local / Access Roads:** The streets right in front of your house that connect to the main highway. | - Provides physical connectivity (ports). <br> - May provide PoE (Power over Ethernet) for phones/APs. <br> - Usually located close to users (e.g., on each floor of a building). |

**Diagram Concept:**
```
[Users] --> [Access Switch] --> [Distribution Switch] --> [Core Switch/Router] --> [Servers/Internet]
```

---

#### The Two-Tier Architecture (Collapsed Core)

This is a simplified design suitable for smaller organizations or campuses that don't need the scale or complexity of a full three-tier model.

| Feature | Description |
| :--- | :--- |
| **Definition** | A two-tier architecture where the **Core** and **Distribution** layers are combined into a single functional layer, often called the **"Collapsed Core."** |
| **Layers** | 1. **Collapsed Core/Distribution** (combined) <br> 2. **Access Layer** |
| **Advantages** | - **Simpler** design and management. <br> - **Less expensive** (fewer devices to purchase and maintain). <br> - Easier to troubleshoot. |
| **Disadvantages** | - **Less resilient** and has fewer redundancy options compared to a full three-tier design. <br> - May not scale as well for very large networks. |

---

#### Data Center Traffic Flow

This terminology describes the direction of data within and around a data center and is critical for security and capacity planning.

| Traffic Type | Direction | Description | Security Implication |
| :--- | :--- | :--- | :--- |
| **North-South Traffic** | **In/Out of the Data Center** | Traffic that enters or leaves the data center. This includes users accessing a web server from the internet (inbound) or a server sending data to an external cloud service (outbound). | **High Scrutiny Required.** This traffic crosses the security perimeter and should be inspected by firewalls, IPS/IDS, etc., as it comes from an untrusted source (the internet). |
| **East-West Traffic** | **Within the Data Center** | Traffic that stays entirely within the data center. For example, a web server communicating with a database server, or a file server transferring data to a backup server. | **Often Overlooked but Critical.** In modern data centers, the majority of traffic is east-west. "Zero Trust" principles dictate that even this internal traffic should be monitored and secured. |

**Diagram Concept:**
- **North-South:** `[Internet] <--> [Firewall] <--> [Data Center]`
- **East-West:** `[Web Server] <--> [Database Server]` (both inside the same data center)
