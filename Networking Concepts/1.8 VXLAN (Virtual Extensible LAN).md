### VXLAN (Virtual Extensible LAN) Cheat Sheet

This guide covers the challenges of traditional VLANs in multi-data center environments and how VXLAN addresses them through encapsulation and tunneling.

---

#### The Problem: Data Center Interconnection (DCI)

As organizations distribute applications across multiple data centers and cloud environments, they face several challenges:

| Challenge | Description |
| :--- | :--- |
| **Different IP Addressing Schemes** | Data Center A might use `10.1.0.0/16`, while Data Center B uses `192.168.0.0/16`. Moving an application between them would require IP address changes unless the networks are bridged. |
| **Different Connectivity Types** | Some data centers have high-speed fiber, others have metro Ethernet or copper. Applications shouldn't need to care about the underlying transport. |
| **VLAN Limitations** | VLANs (802.1Q) have a maximum of **4,094** network identifiers and are limited to **Layer 2**, which cannot be routed across a Layer 3 network (like the internet). |

---

#### The Solution: VXLAN

**VXLAN (Virtual Extensible LAN)** is a network virtualization technology that creates logical Layer 2 networks across a physical Layer 3 infrastructure. It was designed to support large-scale, multi-tenant environments like cloud data centers.

| VXLAN Feature | Benefit |
| :--- | :--- |
| **16 Million Virtual Networks** | Uses a 24-bit VXLAN Network Identifier (VNI), supporting up to **16,777,216** unique networks—vastly more than VLANs' 4,094. |
| **Layer 2 over Layer 3** | Encapsulates original Layer 2 frames inside UDP/IP packets, allowing them to be **routed across any IP network** (including the internet). |
| **Workload Mobility** | Virtual machines (VMs) can move between data centers without changing their IP address or MAC address, enabling true workload portability. |
| **Multi-Tenancy** | Different customers (or applications) can have their own isolated VXLAN segments within the same shared infrastructure. |

---

#### VXLAN Components

| Component | Full Name | Description |
| :--- | :--- | :--- |
| **VTEP** | **VXLAN Tunnel Endpoint** | The device (physical switch, virtual switch, or hypervisor) that encapsulates and decapsulates VXLAN traffic. Each VTEP has a unique IP address. |
| **VNI** | **VXLAN Network Identifier** | A 24-bit identifier (0–16,777,215) that defines a unique VXLAN segment. Similar to a VLAN ID but with much larger scale. |
| **VXLAN Tunnel** | — | An overlay tunnel that carries encapsulated traffic between VTEPs over an IP underlay network. |

---

#### How VXLAN Works: Encapsulation

VXLAN uses **MAC-in-UDP encapsulation** to transport Ethernet frames across a Layer 3 network.

```
Original Frame (in Data Center A):
+-----------------+-----------------+-----------------+
| Ethernet Header | IP Header (VM)  | Payload         |
+-----------------+-----------------+-----------------+

VXLAN Encapsulated Frame (on the wire):
+-----------------+-----------------+-----------------+-----------------+-----------------+
| Outer Ethernet  | Outer IP (VTEP) | UDP (Port 4789) | VXLAN Header    | Original Frame  |
| Header          | Header          |                 | (with VNI)      | (from above)    |
+-----------------+-----------------+-----------------+-----------------+-----------------+
```

| Encapsulation Layer | Purpose |
| :--- | :--- |
| **Outer Ethernet + IP Header** | Routes the packet between VTEPs across the physical network. Uses the VTEP IP addresses (e.g., `1.1.1.1` and `2.2.2.2`). |
| **UDP Header** | VXLAN uses **UDP port 4789** (by default) as the transport. |
| **VXLAN Header** | Contains the 24-bit **VNI** (e.g., VNI 2000, 3000, 4000), identifying the specific virtual network segment. |
| **Original Frame** | The complete original Ethernet frame (including source/destination MAC, original IP header, and payload) is preserved inside the tunnel. |

---

#### VXLAN in Action: Connecting Two Data Centers

| Step | Description |
| :--- | :--- |
| **1. Virtual Machines** | Data Center A has VMs A1, B1, C1. Data Center B has VMs A2, B2, C2. <br> - A1 and A2 are on VNI 2000 <br> - B1 and B2 are on VNI 3000 <br> - C1 and C2 are on VNI 4000 |
| **2. VTEPs** | Each data center has a VTEP (often a top-of-rack switch or virtual switch) with an IP address (e.g., `1.1.1.1` for DC-A, `2.2.2.2` for DC-B). |
| **3. Communication** | When A1 (in DC-A) wants to communicate with A2 (in DC-B): <br> - A1 sends a standard Ethernet frame to its local VTEP. <br> - The VTEP encapsulates the frame with VXLAN headers using VNI 2000. <br> - The encapsulated packet is routed across the IP network to the remote VTEP. <br> - The remote VTEP decapsulates the packet and delivers the original Ethernet frame to A2. |
| **4. Transparency** | To the VMs, they appear to be on the same Layer 2 network, even though they are physically in different data centers. |

---

#### VXLAN vs. VLAN

| Feature | VLAN (802.1Q) | VXLAN |
| :--- | :--- | :--- |
| **Maximum Networks** | 4,094 | 16,777,216 (16 million) |
| **Scope** | Layer 2 only | Layer 2 over Layer 3 (routable) |
| **Encapsulation** | Adds 4-byte tag to Ethernet frame | MAC-in-UDP encapsulation |
| **Transport** | Switched (cannot cross router boundaries) | Routed (can traverse public internet) |
| **Use Case** | Small to medium enterprise networks | Large-scale data centers, cloud providers, multi-tenant environments |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **VXLAN Solves VLAN Limitations** | It provides vastly more network segments (16M vs. 4K) and enables Layer 2 extension over Layer 3 networks. |
| **Workload Mobility** | VMs can move between data centers without IP/MAC changes, enabling seamless migration and disaster recovery. |
| **Overlay Technology** | VXLAN creates an **overlay network** on top of a physical **underlay network**, decoupling the virtual network from the physical infrastructure. |
| **Encapsulation is Key** | By wrapping the original frame in a UDP/IP packet, VXLAN makes traditional Layer 2 traffic routable across the internet. |
