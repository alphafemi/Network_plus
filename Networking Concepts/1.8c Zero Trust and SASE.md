This provides an excellent introduction to the concepts of **Zero Trust** and **SASE (Secure Access Service Edge)** , two of the most important security architectures in modern networking. It explains the shift from traditional perimeter-based security to a model where trust is never assumed and security is delivered from the cloud.



---

### Zero Trust and SASE Cheat Sheet

This guide covers the principles of Zero Trust security and how SASE delivers a unified security and networking solution from the cloud.

---

#### The Problem: Traditional Perimeter Security

| Traditional Model | Challenge |
| :--- | :--- |
| **Hard outer shell, soft interior** | Strong firewalls at the network edge, but once inside, users have broad access to internal resources. |
| **"Trust but verify"** | Users on the internal network are implicitly trusted. |
| **Centralized data centers** | Security was built around protecting a single corporate data center. |

**The Issue:** With users working from anywhere (home, mobile, remote offices) and applications moving to the cloud (SaaS, IaaS), the traditional perimeter no longer exists. A user at a coffee shop is no more "outside" than a user in the corporate office when both are accessing cloud applications.

---

#### Zero Trust Security

Zero Trust is a holistic security framework that operates on a simple principle: **never trust, always verify.**

| Principle | Description |
| :--- | :--- |
| **No Implicit Trust** | Every user, device, and application is considered untrusted until proven otherwise, regardless of location (inside or outside the corporate network). |
| **Verify Every Request** | Every access request is fully authenticated, authorized, and encrypted before granting access. |
| **Least Privilege Access** | Users are granted only the minimum access necessary to perform their job functions—no more. |

---

#### Key Zero Trust Components

| Component | Description |
| :--- | :--- |
| **Adaptive Identity** | Authentication policies that consider context: <br> - Who is the user? (employee, contractor, vendor) <br> - Where are they authenticating from? (geographic location) <br> - What is the device? (corporate-managed or personal) <br> - What time is it? (normal business hours or unusual) |
| **Policy-Based Authentication** | Authentication requirements adapt based on risk. <br> - **Low risk** (on-site, normal hours, corporate device): username + password. <br> - **High risk** (foreign country, VPN, unusual time): additional factors (MFA) or access denied. |
| **Least Privilege Access** | Users receive only the permissions required for their role. <br> - Help desk: read-only access to hardware database. <br> - Help desk manager: modify access to hardware database. <br> - No unnecessary administrative rights (reduces malware impact). |

**Why Least Privilege Matters:**
If a user with administrative privileges clicks a malicious link, the malware inherits those admin rights and can compromise the entire system. Limiting privileges limits the blast radius of an attack.

---

#### What is SASE (Secure Access Service Edge)?

**SASE** (pronounced "sassy") is a cloud-delivered architecture that combines **networking** and **security** capabilities into a single, unified service. It was developed by Gartner to address the needs of modern, distributed workforces.

| Aspect | Description |
| :--- | :--- |
| **Purpose** | Provides secure, optimized access to applications regardless of where users or applications are located. |
| **Delivery** | Delivered from the cloud, not from on-premises appliances. |
| **Deployment** | A lightweight SASE client is installed on every user device. |
| **User Experience** | Transparent to the user—they simply connect to their applications; security is built into the connection. |

---

#### SASE Components (Networking + Security)

SASE combines multiple functions into a single integrated service.

| Function | Description |
| :--- | :--- |
| **Network as a Service** | Includes SD-WAN capabilities, quality of service (QoS), routing, and WAN optimization. |
| **Zero Trust Network Access (ZTNA)** | Provides application-level access based on Zero Trust principles (replaces traditional VPN). |
| **Firewall as a Service (FWaaS)** | Cloud-delivered firewall that inspects all traffic regardless of location. |
| **Secure Web Gateway (SWG)** | Protects users from web-based threats and enforces acceptable use policies. |
| **Cloud Access Security Broker (CASB)** | Secures access to SaaS applications (like Office 365, Salesforce). |
| **DNS Security** | Blocks access to malicious domains at the DNS layer. |

---

#### SASE vs. Traditional VPN

| Feature | Traditional VPN | SASE |
| :--- | :--- | :--- |
| **Architecture** | Hub-and-spoke (backhaul all traffic to corporate data center) | Cloud-delivered (direct to application) |
| **Security** | Separate appliances (firewalls, proxies) at central site | Integrated security stack in the cloud |
| **Performance** | Latency from backhauling traffic | Optimized routing to nearest cloud PoP |
| **Scalability** | Limited by central appliance capacity | Elastic, cloud-native scale |
| **User Experience** | User must manually connect VPN | Transparent; always-on security client |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Zero Trust is a Mindset** | Never assume trust based on location. Verify every access request using context (user, device, location, time). |
| **Least Privilege Reduces Risk** | Limiting user permissions limits the damage potential of compromised accounts. |
| **SASE Delivers Security from the Cloud** | Combines networking (SD-WAN) and security (ZTNA, FWaaS, SWG, CASB) into a single, cloud-native service. |
| **SASE Supports Anywhere, Any Device** | With a lightweight client, users get secure access to applications regardless of where they are or where the apps reside. |
| **Transparent Security** | SASE operates in the background—users don't need to toggle VPNs or remember security settings. |
