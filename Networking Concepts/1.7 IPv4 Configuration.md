#### Core IPv4 Configuration

Every device needs these three basic settings to communicate on an IP network.

| Component | Example | Description |
| :--- | :--- | :--- |
| **IP Address** | `192.168.1.165` | A unique 32-bit identifier for a device on a network. It is a **logical address** (OSI Layer 3). |
| **Subnet Mask** | `255.255.255.0` | A 32-bit value that works with the IP address to determine which part of the address is the **network** and which part is the **host**. It tells the device what subnet it belongs to. |
| **Default Gateway** | `192.168.1.1` | The IP address of the router on the local subnet. This is the "exit door" that allows a device to communicate with devices on **other networks** (like the internet). |

**Key Point:** The Subnet Mask and Default Gateway are used by the *local device* to make forwarding decisions. The Subnet Mask is not usually transmitted across the network with every packet.

---

#### Anatomy of an IPv4 Address

| Concept | Description |
| :--- | :--- |
| **Structure** | Four decimal numbers separated by periods (dots): `192.168.1.131`. |
| **Octet** | Each of the four numbers is called an **octet** because it represents 8 bits. |
| **32-bit Address** | An IPv4 address is a 32-bit binary number, grouped into four 8-bit octets. |
| **Maximum Value** | Since each octet is 8 bits, its maximum decimal value is **255** (binary `11111111`). |

---

#### Special IPv4 Address Types

| Address Type | Range / Example | Description / Use Case |
| :--- | :--- | :--- |
| **Loopback** | `127.0.0.1` - `127.255.255.254` | A special address used to test a device's own internal IP stack. `127.0.0.1` is the most common. It never appears on the physical network. |
| **Reserved (Class E)** | `240.0.0.1` - `254.255.255.254` | A range of addresses reserved for future use or testing. They should not be assigned to devices. |
| **Virtual IP (VIP)** | (Any assigned IP) | An IP address not tied to a physical network interface. Used for redundancy and failover (e.g., on a router cluster, load balancer, or virtual machine). |

---

#### Automatic IP Assignment

| Method | Protocol | Description | Address Range |
| :--- | :--- | :--- | :--- |
| **DHCP** | **Dynamic Host Configuration Protocol** | The standard way devices get IP configuration automatically when they connect to a network. It provides the IP address, subnet mask, gateway, DNS, and more. | Assigned from a pool configured by the administrator. |
| **APIPA**<br> (Link-Local) | **Automatic Private IP Addressing** | A fallback mechanism when a DHCP server is unavailable. The device assigns itself an address. It can only communicate with other devices on the **same local subnet** (cannot route to the internet). | **`169.254.0.1` - `169.254.255.254`** <br> (with the first/last 256 addresses reserved). |

#### Private IP Addressing (RFC 1918)

Because public IPv4 addresses are exhausted, RFC 1918 reserves specific ranges for private use. These addresses **cannot be routed on the public internet**.

| Network Range | CIDR Notation | Subnet Mask | Total Addresses | Description |
| :--- | :--- | :--- | :--- | :--- |
| **10.0.0.0 – 10.255.255.255** | **10.0.0.0/8** | `255.0.0.0` | ~16.7 Million | One large "Class A" network. |
| **172.16.0.0 – 172.31.255.255** | **172.16.0.0/12** | `255.240.0.0` | ~1 Million | 16 contiguous "Class B" networks. |
| **192.168.0.0 – 192.168.255.255** | **192.168.0.0/16** | `255.255.0.0` | ~65,000 | 256 contiguous "Class C" networks. |

**The NAT Solution:**
Private IP addresses allow devices inside a network to communicate with each other. To access the internet, a device like a router or firewall performs **Network Address Translation (NAT)** , which converts the private IP address to a public, routable IP address.
