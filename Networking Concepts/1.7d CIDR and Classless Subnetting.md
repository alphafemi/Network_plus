This  provides a clear and practical introduction to **Classless Inter-Domain Routing (CIDR)** , which is the modern standard for subnetting. It explains how to move beyond the rigid classful system (A, B, C) by using a flexible notation that defines the subnet mask by the number of bits used for the network portion.



### CIDR and Classless Subnetting Cheat Sheet

This guide covers how CIDR notation works, how to convert between binary, decimal, and CIDR formats, and how to interpret the subnet mask.

---

#### What is CIDR (Classless Interdomain Routing)?

| Concept | Description |
| :--- | :--- |
| **Classless Subnetting** | A system introduced in 1993 that replaced the rigid classful system (Class A, B, C). It allows you to use any subnet mask, not just the default ones (`255.0.0.0`, `255.255.0.0`, `255.255.255.0`). |
| **CIDR Notation** | A shorthand way to write the subnet mask. Instead of writing `255.255.255.0`, you write **`/24`**. The number after the slash is the count of contiguous `1` bits in the subnet mask. |
| **Example** | `192.168.1.44/24` means the IP address is `192.168.1.44` with a subnet mask of `/24` (which is `255.255.255.0`). |

---

#### The Subnet Mask Structure

A subnet mask is always a contiguous string of **`1`** bits followed by **`0`** bits.

| Subnet Mask (Binary) | Number of `1` Bits (Network) | Number of `0` Bits (Host) | CIDR Notation | Decimal Mask |
| :--- | :--- | :--- | :--- | :--- |
| `11111111.00000000.00000000.00000000` | 8 | 24 | **/8** | `255.0.0.0` |
| `11111111.11111111.00000000.00000000` | 16 | 16 | **/16** | `255.255.0.0` |
| `11111111.11111111.11111111.00000000` | 24 | 8 | **/24** | `255.255.255.0` |

---

#### Converting Between Decimal and CIDR

Because the subnet mask is contiguous, you will only see specific decimal values in each octet. This chart is essential for conversion.

| Bit Pattern (Binary) | Decimal Value | CIDR Equivalent (for that octet) |
| :--- | :--- | :--- |
| `00000000` | **0** | - |
| `10000000` | **128** | /1 |
| `11000000` | **192** | /2 |
| `11100000` | **224** | /3 |
| `11110000` | **240** | /4 |
| `11111000` | **248** | /5 |
| `11111100` | **252** | /6 |
| `11111110` | **254** | /7 |
| `11111111` | **255** | /8 |

---

#### Example Conversions

**1. Binary Subnet Mask to CIDR**

Given a binary mask: `11111111.11111111.11111111.11000000`

- Count the `1` bits: 8 + 8 + 8 + 2 = **26**
- CIDR Notation: **`/26`**
- Decimal Equivalent: `255.255.255.192`

**2. CIDR Notation to Decimal**

Given CIDR: **`/20`**

- This means 20 `1` bits.
- The first 16 bits are two full octets: `11111111.11111111` = `255.255`
- The next 4 bits are in the third octet: `11110000` = **240**
- The fourth octet has 0 `1` bits: `00000000` = 0
- Decimal Subnet Mask: **`255.255.240.0`**

**3. CIDR Notation to Decimal**

Given CIDR: **`/26`**

- This means 26 `1` bits.
- The first 24 bits are three full octets: `11111111.11111111.11111111` = `255.255.255`
- The remaining 2 bits are in the fourth octet: `11000000` = **192**
- Decimal Subnet Mask: **`255.255.255.192`**

---

#### Key Takeaways

| Concept | Description |
| :--- | :--- |
| **Network ID Bits** | The number after the `/` tells you exactly how many bits are used for the network portion of the address. |
| **Host ID Bits** | The remaining bits (32 minus the CIDR number) are for host addresses. |
| **Device Configuration** | Some devices (like Windows) expect the decimal subnet mask (e.g., `255.255.255.0`), while others (like routers and switches) may expect CIDR notation (e.g., `/24`). Always check the documentation. |
| **Contiguous Rule** | The subnet mask must always be a contiguous block of `1`s. You cannot have a mask like `11101111.00000000.00000000.00000000`. |
