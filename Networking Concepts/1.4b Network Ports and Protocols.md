Network Ports and Protocols
As a networking professional, you will often be asked to configure port numbers associated with specific applications. Understanding these ports is essential for network security, troubleshooting, and configuration. Below is a comprehensive guide to the well-known port numbers and their associated protocols.

File Transfer Protocols
File transfer protocols allow data to move between devices. Some are secure, while others transmit data in plain text.

FTP (File Transfer Protocol)

Ports: TCP 20 and TCP 21
Function: A generic form of file transfer used across many operating systems (Linux, Windows, Mac OS).
Details: TCP port 21 is used for control information and authentication (usernames/passwords). TCP port 20 is used for the actual file transfer process.
Capabilities: Allows listing directories, adding, deleting, renaming, and modifying files.
SFTP (Secure File Transfer Protocol)

Port: TCP 22
Function: A secure version of FTP that encrypts all communication by default.
Details: It uses the SSH protocol to perform encryption. It allows the same file management capabilities as FTP but over a secure connection.
TFTP (Trivial File Transfer Protocol)

Port: UDP 69
Function: Used for very simple file transfers that do not require authentication.
Details: Often used to transfer configuration files to devices like Voice over IP phones. It transfers data quickly without extra overhead.
Remote Access and Management
These protocols allow administrators to manage devices remotely. Security is a major differentiator between them.

SSH (Secure Shell)

Port: TCP 22
Function: Allows communication to a remote device from a console using a text-based command line interface.
Security: All communication is encrypted. It is the preferred method for remote management.
Note: SFTP also uses this port because it relies on the SSH protocol foundation.
Telnet

Port: TCP 23
Function: A non-encrypted form of terminal communication.
Security: Communication is sent in the clear. Login credentials can be captured by attackers.
Usage: Rarely used today due to security risks; SSH is preferred.
RDP (Remote Desktop Protocol)

Port: TCP 3389
Function: Allows viewing the remote desktop of a device.
Details: Commonly associated with Windows devices but has clients for Mac OS, Linux, iPhone, and Android.
Web and Email Services
These protocols handle web browsing and email transmission.

HTTP (Hypertext Transfer Protocol)

Port: TCP 80
Function: Browser-based communication sent in the clear without encryption.
Usage: Common for older or unsecured websites.
HTTPS (HTTP Secure)

Port: TCP 443
Function: Encrypted web communication using SSL or TLS.
Usage: The default for most modern websites to ensure data security.
SMTP (Simple Mail Transfer Protocol)

Ports: TCP 25 and TCP 587
Function: Used for server-to-server email transfers and sending email from client devices.
Security: Port 25 sends traffic in plain text. Port 587 is used for SMTP with TLS encryption.
IMAP and POP3

Function: Protocols used for receiving email messages and managing the inbox.
Usage: Often used in conjunction with SMTP for full email functionality.
Network Configuration and Resolution
These protocols handle automatic IP assignment and name resolution.

DNS (Domain Name System)

Ports: UDP 53 and TCP 53
Function: Translates Fully Qualified Domain Names (FQDN) to IP addresses.
Details: UDP 53 is used for small queries. TCP 53 is used for large data transfers between DNS servers.
DHCP (Dynamic Host Configuration Protocol)

Ports: UDP 67 and UDP 68
Function: Automatically configures IP address settings for devices connecting to the network.
Details: Assigns IP addresses from a pool with a specific lease time. Can also create reservations based on MAC addresses.
Network Management and Monitoring
These protocols help administrators monitor device health and collect logs.

NTP (Network Time Protocol)

Port: UDP 123
Function: Synchronizes clocks across all devices on the network.
Importance: Critical for synchronizing log files to piece together communications across diverse devices.
SNMP (Simple Network Management Protocol)

Ports: UDP 161 and UDP 162
Function: Manages network devices like routers, switches, and firewalls.
Details:
UDP 161: Used to query devices for performance data (e.g., traffic volume).
UDP 162: Used for SNMP traps, where devices proactively send notifications about problems.
Versions: Version 3 is the most secure, offering encryption and authentication.
Syslog

Port: UDP 514
Function: A standardized protocol for transferring log files to a single location.
Usage: Often used with a Security Information and Event Manager (SIEM) to consolidate logs from various devices.
Directory and Database Services
These protocols manage user information and database access.

LDAP (Lightweight Directory Access Protocol)

Ports: TCP 389 and TCP 636
Function: Maintains a database of devices and users in a hierarchical structure.
Details: TCP 389 is the standard port. TCP 636 is used for LDAP Secure (encrypted).
Structure: Organizes data by organization, organizational units, and common names.
MS-SQL (Microsoft Structured Query Language)

Port: TCP 1433
Function: A standard language for querying and retrieving information from databases.
Details: Specifically refers to Microsoft SQL Server. Other SQL databases may use different ports.
SMB (Server Message Block)

Port: TCP 445
Function: Used for sharing files, printers, and authentication on Microsoft Windows networks.
Details: Also known as CIFS (Common Internet File System). Integrated into Windows Explorer.
Note: Older versions used NetBIOS, but modern Windows uses IP directly on port 445.
Voice over IP (VoIP)
SIP (Session Initiation Protocol)
Ports: TCP 5060 and TCP 5061
Function: A control protocol for initiating, maintaining, and disconnecting phone calls.
Details: Used for voice calls, video conferencing, instant messaging, and file transfers over IP networks.
