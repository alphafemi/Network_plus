This  provides a clear and concise explanation of how **IPv6 address autoconfiguration** works without a DHCP server. It introduces the **Neighbor Discovery Protocol (NDP)** and explains the **Stateless Address Autoconfiguration (SLAAC)** process, along with how duplicate addresses are prevented and how routers are discovered.


---

### IPv6 Autoconfiguration (SLAAC and NDP) Cheat Sheet

This guide covers how IPv6 devices can automatically configure themselves without a DHCP server, using the Neighbor Discovery Protocol and Stateless Address Autoconfiguration.

---

#### IPv6 Addressing: Two Approaches

| Method | Description | Management | Lease | Use Case |
| :--- | :--- | :--- | :--- | :--- |
| **Stateful (DHCPv6)** | Traditional DHCP server assigns addresses and tracks leases. | Administrator manages DHCPv6 server. | Yes (lease time) | Enterprise environments requiring central control and logging. |
| **Stateless (SLAAC)** | Device creates its own IPv6 address without a server. | No server; device self‑manages. | No lease (permanent until device leaves) | Simpler networks; IoT; networks without DHCPv6 infrastructure. |

---

#### Stateless Address Autoconfiguration (SLAAC)

SLAAC allows an IPv6 device to create a globally routable IPv6 address completely on its own, without contacting a DHCP server.

**The SLAAC Process (Two Parts):**

| Step | Component | Description |
| :--- | :--- | :--- |
| **1** | **Subnet Prefix (first 64 bits)** | Device learns the local subnet prefix from a router using NDP (Router Solicitation / Router Advertisement). |
| **2** | **Interface ID (last 64 bits)** | Device creates the remaining 64 bits, either from its MAC address (EUI‑64 format) or a random value. |

**Result:** A complete 128‑bit IPv6 address.

---

#### Neighbor Discovery Protocol (NDP)

NDP is the IPv6 replacement for several IPv4 protocols, including ARP (Address Resolution Protocol) and parts of ICMP. It uses **multicast** instead of broadcast, making it more efficient.

| NDP Function | Description | IPv4 Equivalent |
| :--- | :--- | :--- |
| **Router Solicitation (RS)** | Device asks "Are there any routers on this subnet?" (multicast). | (No direct equivalent) |
| **Router Advertisement (RA)** | Router responds with subnet prefix, prefix length, DNS, and other settings. Can be unicast (in response to RS) or unsolicited (periodic multicast). | (No direct equivalent) |
| **Duplicate Address Detection (DAD)** | Device checks whether its self‑assigned address is already in use on the network. | (ARP‑based duplicate detection) |
| **Neighbor Solicitation / Advertisement** | Finds the link‑layer (MAC) address of another IPv6 device (replaces ARP). | ARP |

---

#### How SLAAC Works (Step‑by‑Step)

| Step | Action | Protocol / Method |
| :--- | :--- | :--- |
| **1. Discover the Subnet** | Device sends a **Router Solicitation** (multicast) asking for local router information. | NDP (RS) |
| **2. Receive Subnet Information** | Router responds with a **Router Advertisement** containing the 64‑bit subnet prefix (e.g., `2001:db8:1234:5678::/64`). | NDP (RA) |
| **3. Generate Interface ID** | Device creates the last 64 bits: <br> - **EUI‑64:** Takes its MAC address, inserts `FF:FE` in the middle, flips the Universal/Local bit. <br> - **Privacy Extensions:** Generates a random 64‑bit value (recommended for privacy). | Built into OS |
| **4. Combine** | Device combines the 64‑bit prefix + 64‑bit interface ID → full IPv6 address. | SLAAC |
| **5. Verify Uniqueness** | Device sends a **Duplicate Address Detection (DAD)** query to ensure no other device is using the same address. | NDP (DAD) |
| **6. Address Ready** | If DAD passes (no duplicate found), the address is assigned and usable. | — |

**EUI‑64 Example (MAC to Interface ID):**
- MAC Address: `00:1A:2B:3C:4D:5E`
- Insert `FF:FE`: `001A:2BFF:FE3C:4D5E`
- Flip the U/L bit (7th bit of the first byte): `021A:2BFF:FE3C:4D5E`

---

#### Duplicate Address Detection (DAD)

| Aspect | Description |
| :--- | :--- |
| **Purpose** | Prevent IP address conflicts when using SLAAC (or even with DHCPv6). |
| **How It Works** | Before finalizing an address, the device sends a Neighbor Solicitation asking, "Does anyone already use this address?" If no response, the address is unique. |
| **Scope** | Local subnet only (like ARP in IPv4). |

---

#### Router Solicitation and Advertisement (RS/RA)

| Message | Direction | Description |
| :--- | :--- | :--- |
| **Router Solicitation (RS)** | Client → Router | Sent by a device when it first connects to a network, asking routers to identify themselves. |
| **Router Advertisement (RA)** | Router → Client | Sent in response to an RS, or periodically as an **unsolicited RA**. Contains: <br> - Subnet prefix and prefix length <br> - DNS server addresses (RDNSS option) <br> - Other configuration parameters |

**Key Point:** RA messages can carry DNS information (RDNSS – Recursive DNS Server option), eliminating the need for DHCPv6 for basic network configuration.

---

#### SLAAC vs. DHCPv6

| Feature | SLAAC (Stateless) | DHCPv6 (Stateful) |
| :--- | :--- | :--- |
| **Server Required** | No (router provides prefix via RA) | Yes (DHCPv6 server) |
| **Address Tracking** | No central lease table | Yes (server logs assignments) |
| **Lease Time** | No lease (address is "permanent" until device leaves) | Yes (configurable lease time) |
| **Additional Configuration** | Limited (RAs can provide DNS, but not all parameters) | Full control (DNS, domain, NTP, SIP server, etc.) |
| **Privacy** | EUI‑64 exposes MAC address; Privacy Extensions (random) solve this | Can assign temporary or stable addresses |
| **Use Case** | Simpler networks, IoT, environments without DHCPv6 | Enterprise networks requiring audit trails and full configuration control |

**Hybrid Approach:** Many networks use SLAAC for address assignment but DHCPv6 for additional options (DNS, domain, etc.). This is sometimes called "stateless DHCPv6."

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **IPv6 Can Work Without DHCP** | SLAAC allows devices to self‑assign addresses using NDP and router advertisements. |
| **NDP Replaces ARP** | NDP uses efficient multicast instead of broadcast for neighbor discovery, router discovery, and duplicate address detection. |
| **SLAAC = Subnet Prefix + Interface ID** | The device learns the prefix from a router (via RA) and generates the last 64 bits (from MAC or random). |
| **DAD Prevents Conflicts** | Before using a self‑assigned address, the device verifies uniqueness with Duplicate Address Detection. |
| **Routers Advertise Subnet Info** | Router Advertisements (RAs) provide the subnet prefix and can also supply DNS server addresses (RDNSS). |
| **SLAAC + DHCPv6 Can Coexist** | Many networks use SLAAC for address assignment and DHCPv6 for additional configuration options. |
