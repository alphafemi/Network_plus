This  provides a thorough explanation of **DHCP configuration** from the server perspective. It covers how IP address pools are defined, how to ensure specific devices always receive the same IP, how lease times work, and how to extend DHCP to configure additional network parameters.



---

### DHCP Server Configuration Cheat Sheet

This guide covers the core components of DHCP server configuration, including scopes, reservations, lease timers, and DHCP options.

---

#### DHCP Scopes

A **DHCP scope** is the fundamental configuration unit on a DHCP server. It defines all the settings that a client on a specific subnet will receive.

| Scope Component | Description | Example |
| :--- | :--- | :--- |
| **IP Address Range** | The pool of addresses the server can assign. Usually a contiguous block. | `165.245.44.1` – `165.245.44.100` |
| **Exclusions** | Addresses within the range that should **not** be handed out (e.g., statically configured servers). | Exclude `.1` – `.10` |
| **Subnet Mask** | The mask for the subnet. | `255.255.255.0` |
| **Lease Duration** | How long a client can keep an assigned IP address. | 8 days (configurable) |
| **Scope Options** | Additional settings provided to clients (DNS, gateway, etc.). | DNS servers, default gateway, VoIP server IP |

**Typical Scope Setup (per subnet):**
- One scope is created for each subnet that the DHCP server serves.
- The server selects an available address from the pool and assigns it to a requesting client.

---

#### Address Reservations (Static DHCP)

A **reservation** ensures that a specific device always receives the same IP address from the DHCP server, without manually configuring the device.

| Aspect | Description |
| :--- | :--- |
| **How It Works** | The DHCP server maintains a table mapping MAC addresses to fixed IP addresses. |
| **Use Cases** | Printers, servers, network appliances—any device that needs a consistent IP but benefits from centralized management. |
| **Other Names** | Static DHCP assignment, static DHCP, IP reservation. |
| **Advantage** | Centralized management. Change DNS or gateway settings in one place (the DHCP server) instead of touching every device. |

**Example Reservation Table:**
| IP Address | Device Name | MAC Address |
| :--- | :--- | :--- |
| `192.168.1.6` | Prometheus | `00:1A:2B:3C:4D:5E` |
| `192.168.1.9` | Odyssey | `00:1A:2B:3C:4D:5F` |

#### DHCP Lease Timers (T1 and T2)

DHCP addresses are **leased**, not permanently assigned. The lease duration is configurable by the administrator. The client attempts to renew the lease before it expires using two timers.

| Timer | Default Setting | Description |
| :--- | :--- | :--- |
| **T1 (Renewal Timer)** | **50%** of lease time | Client attempts to **renew the lease** with the **original DHCP server**. |
| **T2 (Rebinding Timer)** | **87.5%** (7/8) of lease time | If T1 failed, client attempts to renew with **any available DHCP server**. |

**Example: 8‑Day Lease**

| Timer | Calculation | Action |
| :--- | :--- | :--- |
| **T1** | 4 days | Client tries to renew with the original DHCP server. |
| **T2** | 7 days | If original server is unavailable, client tries any DHCP server. |
| **Lease Expiration** | 8 days | If no renewal, IP address returns to the pool. |

**Normal Operation:**
1. Client receives lease (8 days).
2. At day 4 (T1), client renews with original server → lease resets to 8 days.
3. Process repeats.

**Failure Scenario:**
1. Original DHCP server fails after day 4.
2. Client continues using the address until day 7 (T2).
3. At day 7, client reaches out to **any** available DHCP server (e.g., a backup server) and renews the lease.

**Graceful Shutdown:** A client can **release** its IP address back to the pool when shutting down, making it immediately available to other devices.

#### DHCP Options

DHCP can configure more than just IP address, subnet mask, and gateway. **DHCP options** (originally called "vendor extensions" in BOOTP) are numbered fields that carry additional configuration parameters.

| Aspect | Description |
| :--- | :--- |
| **Number of Options** | 254 usable options (defined in RFCs). |
| **Option Format** | Each option has a number and a value (e.g., Option 6 = DNS servers). |
| **Common Options** | Subnet mask (1), default gateway (3), DNS servers (6), domain name (15). |
| **Advanced Options** | Option 129 = VoIP call server IP address; Option 135 = HTTP proxy. |

**Common DHCP Options:**

| Option Number | Description |
| :--- | :--- |
| **1** | Subnet Mask |
| **3** | Default Gateway (Router) |
| **6** | DNS Servers |
| **15** | Domain Name |
| **51** | Lease Time |
| **66** | TFTP Server Name (used for VoIP phone boot) |
| **67** | Bootfile Name |
| **129** | VoIP Call Server IP Address |
| **135** | HTTP Proxy |

**Important Note:** Not all DHCP servers support all options. Check your server's documentation before configuring advanced options.

#### Putting It All Together: Scope Configuration (Conceptual)

A typical DHCP scope configuration includes:

| Setting | Example Value |
| :--- | :--- |
| **Scope Name** | `Corporate LAN` |
| **IP Range** | `192.168.1.10` – `192.168.1.200` |
| **Exclusions** | `192.168.1.1` – `192.168.1.9` (reserved for static devices) |
| **Subnet Mask** | `255.255.255.0` |
| **Lease Duration** | `8 days` |
| **Default Gateway (Option 3)** | `192.168.1.1` |
| **DNS Servers (Option 6)** | `8.8.8.8`, `1.1.1.1` |
| **Domain Name (Option 15)** | `example.com` |
| **Reservations** | MAC → IP mappings for printers, servers |

---

#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **Scopes Define Address Pools** | Each subnet requires its own scope on the DHCP server. |
| **Exclusions Prevent Conflicts** | Reserved for statically configured devices (routers, critical servers). |
| **Reservations Ensure Consistent IPs** | Devices get the same IP every time without manual configuration. |
| **Lease Timers Control Renewal** | T1 (50%) renews with original server; T2 (87.5%) rebinds with any server. |
| **DHCP Options Extend Configuration** | Can assign DNS, gateway, VoIP servers, proxies, and more. |
| **Centralized Management** | DHCP allows you to manage IP configuration from a single server rather than touching every device. |
