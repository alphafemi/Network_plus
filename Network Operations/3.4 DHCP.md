It also explains the limitation of DHCP broadcasts and how a **DHCP relay** solves that problem.

### DHCP (Dynamic Host Configuration Protocol) Cheat Sheet

This guide covers the evolution from manual configuration to DHCP, the DORA process, and how DHCP relay enables centralized DHCP servers across different subnets.

---

#### The Evolution of IP Configuration

| Method | Description | Limitation |
| :--- | :--- | :--- |
| **Manual Configuration** | Administrator configures IP address, subnet mask, DNS, etc., on every device. | Does not scale; every change requires touching every device. |
| **BOOTP (Bootstrap Protocol)** | Early automated configuration protocol. | Did not handle dynamic address reuse (addresses weren't reclaimed when devices left). |
| **DHCP (Dynamic Host Configuration Protocol)** | Modern protocol that automates IP configuration. | Dynamically assigns, tracks, and reclaims addresses; handles all TCP/IP parameters. |

---

#### The DORA Process

DHCP uses a four‑step process to assign an IP address. The acronym **DORA** helps remember the steps.

| Step | Name | Direction | Key Characteristics |
| :--- | :--- | :--- | :--- |
| **D** | **Discover** | Client → Server | - Client sends broadcast to find DHCP servers. <br> - Source IP: `0.0.0.0` (no IP yet) <br> - Destination: `255.255.255.255` (broadcast) <br> - UDP source port: **68** / destination port: **67** |
| **O** | **Offer** | Server → Client | - DHCP server offers an available IP address. <br> - Sent as broadcast because client has no IP. <br> - UDP source port: **67** / destination port: **68** |
| **R** | **Request** | Client → Server | - Client requests the offered address. <br> - Still uses broadcast so all DHCP servers know the client has chosen one. <br> - Source IP: `0.0.0.0` / Destination: `255.255.255.255` |
| **A** | **Acknowledgment** | Server → Client | - DHCP server acknowledges and finalizes the lease. <br> - Client can now configure its IP stack. <br> - Sent as broadcast |

---

#### DHCP Ports

| Role | UDP Port |
| :--- | :--- |
| **DHCP Client** | **UDP 68** |
| **DHCP Server** | **UDP 67** |

#### The DHCP Broadcast Problem

| Issue | Explanation |
| :--- | :--- |
| **Broadcast Domain Limitation** | Routers do not forward broadcast traffic (`255.255.255.255`) by default. |
| **Problem** | If a DHCP server is on a different subnet, a client's DHCP Discover broadcast will never reach it. |
| **Impact** | Without additional configuration, every subnet must have its own DHCP server. |

---

#### Solution: DHCP Helper / DHCP Relay

A **DHCP Relay** (also called DHCP Helper) is a router feature that forwards DHCP traffic between subnets, converting broadcasts to unicasts and back.

**How It Works (Client on Remote Subnet, Server at `10.10.10.99`):**

| Step | Description |
| :--- | :--- |
| **1. Discover** | Client sends DHCP Discover broadcast on its local subnet. |
| **2. Relay Receive** | Router (configured with DHCP relay) receives the broadcast. |
| **3. Relay Transform** | Router changes the packet: <br> - Source IP → router's own IP (e.g., `10.10.30.1`) <br> - Destination IP → DHCP server IP (`10.10.10.99`) <br> - Converts broadcast → **unicast** |
| **4. Forward** | Router forwards the unicast packet to the DHCP server across the network. |
| **5. Offer** | DHCP server sends Offer back to the router's IP address (unicast). |
| **6. Relay Reverse** | Router converts Offer back to broadcast and sends it on the local subnet. |
| **7. Request & Acknowledge** | Same process repeats for the Request and Acknowledge steps. |

**Key Benefit:** Allows centralized DHCP servers to serve multiple subnets, reducing administrative overhead and enabling redundancy.

---



#### Key Takeaways

| Takeaway | Explanation |
| :--- | :--- |
| **DHCP Automates IP Configuration** | Replaced manual configuration and BOOTP with a dynamic, scalable solution. |
| **DORA is the Four-Step Process** | Discover → Offer → Request → Acknowledgment. |
| **DHCP Uses Broadcasts** | Clients send broadcasts to locate servers because they have no IP address yet. |
| **Broadcasts Do Not Cross Routers** | A DHCP server on a different subnet cannot be reached by a client's broadcast. |
| **DHCP Relay Solves This** | Converts broadcast DHCP messages to unicasts and forwards them to a remote DHCP server. |
| **Centralized DHCP** | Relay allows a single DHCP server (or server cluster) to serve the entire enterprise. |
